## Locked-In Postures

## 1. Global Absorption Comes First

Do not treat database absorption as the whole story.

The broad absorption posture is now locked in:

```text
The bare factorization failure is old.
```

Primary absorbers include:

- statistical decision theory;
- Blackwell comparison of experiments;
- Le Cam deficiency;
- concurrency and process semantics;
- van Glabbeek's linear-time/branching-time spectrum;
- testing equivalences;
- bisimulation;
- Hennessy-Milner-style logics;
- abstract interpretation;
- complete abstractions;
- POMDP belief-state sufficiency;
- resource-theoretic convertibility;
- causal observational/interventional equivalence;
- control observability, controllability, and reachability;
- database view/query determinacy;
- provenance and lineage;
- quantum information and resource theory.

Future reviewers should ask:

```text
Which mature theory already owns this version of the claim?
```

before asking:

```text
Is this a new Candidate North Star result?
```

## 2. Database Theory Is A Primary Absorber And Finite-Test Source

Database systems are not just another example. They are one of the cleanest
ways to make the factorization test finite and executable.

Locked-in database translation:

```text
Y = full database state
pi = view, projection, materialized snapshot, index, API surface, or visible result
Cap = query, workload, recovery operation, audit right, update right,
      retrieval behavior, or policy-evaluable object
```

Database examples should usually be framed as:

- finite witnesses;
- preservation controls;
- same-visible-state discipline checks;
- translation residue;
- heuristic residue.

They should not be framed as formal novelty unless they survive both:

- database-native state enrichment;
- database-native theory absorption.

## 3. Strong Physics Stance Is Locked In

The physics sections should not be softened into safety labels.

Locked-in posture:

```text
be bold, but not incorrect
refine, but do not soften
physics as typed strongman, not mere analogy padding
```

Each physics-facing section should instantiate:

```text

```

Physics remains:

- non-canonical;
- non-promotional;
- not evidential support;
- not a replacement for known physics.

But it should still make the strongest mathematically typed case that known
physics already contains projection, fiber, accessibility, and capability-like
structures.

Additional locked-in physics rule:

```text
known physics -> induced capability object -> projection sufficiency audit
```

Do not reverse this into:

```text
capability object -> known physics
```

unless future theorems actually justify that stronger direction.

Physics-facing claims must derive `Cap` from established physical theory,
allowed operations, constraints, observers, and boundaries.

## 4. Capability Type Discipline Comes Before Examples

No example should be discussed before declaring `K`.

Allowed `K` types include:

- set of answers;
- set of admissible operations;
- preorder of admissible future operations;
- decision-value, risk, or loss object;
- provenance or lineage object;
- behavioral equivalence class;
- testing capability or distinguishing formula class;
- resource convertibility structure;
- reachable or viable set;
- causal intervention query class;
- database workload or rewritability object;
- approximate retrieval envelope;
- physics-facing accessibility structure.

Locked-in rule:

```text
Capability equivalence must be domain-native, task-natural, and fixed before
witness construction.
```

## 5. The Finite Pair Test Is Necessary But Weak

The finite pair test:

```text
pi(y1) = pi(y2)
Cap(y1) not ~=_K Cap(y2)
```

proves only:

```text
the chosen projection does not determine the chosen capability
```

It does not prove:

```text
novelty
deep hidden structure
formal residue
```

Future reviewers should ask:

- Is `Cap` domain-natural?
- Is the projection operationally meaningful?
- Is the task family non-gerrymandered?
- Is equivalence on `K` justified?
- Did obvious enrichment restore factorization?
- Does native prior art already explain the failure?

## 6. Canonicity Is The Hard Target

The strongest target is now:

```text
canonical_residue
```

Use it only when:

- the witness survives same-neighbor-data testing;
- the witness survives native-theory absorption;
- `Cap` is domain-natural or operationally forced;
- the projection is not obviously impoverished;
- the result reveals an unavoidable tradeoff across observer, task, horizon,
  and resource boundary.

Failure label:

```text
gerrymandered_capability
```

Use it when `Cap` is technically predeclared but not operationally natural,
canonical, or native to the domain.

## 7. Absorption Has Two Forms

Future reviews must preserve this distinction.

### State-Enrichment Absorption

The neighbor's legitimate state distinguishes the cases, restores
factorization, or shows that the original projection was underdescribed.

### Native-Theory Absorption

The neighbor already has the theorem, schema, equivalence hierarchy, or
completeness criterion explaining the non-factorization.

Formal or canonical residue must survive both.

## 8. Same-Visible-State Discipline Is Mandatory

Failure label:

```text
same_visible_state_underspecified
```

Use it when a witness claims same visible state without declaring the relevant
context.

- valid time;
- isolation level;

- provenance/lineage regime;
- index/materialization state;
- access policy;
- consistency model;
- causal metadata;
- embedding model;
- distance metric;

Physics context may include:

- local patch or causal domain;
- measurement interface;
- global versus local state;
- allowed operations;
- horizon;
- resource constraints;
- equivalence on future operations.

```text
by the neighboring field, record absorption, translation residue, or heuristic
residue. Do not promote the witness as formal residue.
```

## C. Physics Strongman Check

Do the physics sections instantiate the schema strongly enough?

For each section, ask:

- Is `Y` domain-native?
- Is `X` a plausible observer-visible projection?
- Is `pi` clear?
- Is `Cap` typed enough?
- Is `K` named?
- Is equivalence on `K` declared?
- Is the absorber honest?
- Is the "not claimed" boundary strong enough?
- Did we preserve ambition?

Also check:

- Is the access profile physically meaningful?
- Does the example avoid smuggling inaccessible global structure into a local
  observer's `Cap`?
- Is `Cap` invariant under gauge or coordinate redundancy?
- Are boundary conditions, asymptotics, environment, and resource constraints
  declared where needed?
- Is the section deriving capability from known physics, not physics from
  capability?

## D. Canonicity Check

Does v0.3 explain when a witness is interesting, or does it still rely too much

- coarsest adequate quotients;
- fully abstract semantics;
- Nerode congruence;
- minimal predictive states;
- complete abstract domains;
- domain-native resource monotones;
- canonical database views or workloads.
