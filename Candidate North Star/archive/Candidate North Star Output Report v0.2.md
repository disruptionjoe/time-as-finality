# Candidate North Star Output Report v0.2

## Status

This is an output report for the next Candidate North Star draft:

```text
Candidate North Star v0.2.md
```

It is not canon.
It does not promote the candidate.
It does not replace `NORTH-STAR.md`.

## What Changed

The new draft shifts the Candidate North Star from a physics-forward intuition
to an operational factorization audit.

The previous draft was centered on:

```text
observable shadow + capability fiber
```

The new draft is centered on:

```text
For fixed observer, task, horizon, and resource boundary,
future capability need not factor through observer-visible state.
```

The strongest new guardrail is:

```text
a non-factorization verdict is a projection-sufficiency failure,
not a metaphysical claim
```

This directly answers the main hostile objection: if the visible state is too
weak, enrich it. Enrichment is allowed and often preferred.

## Main Design Decisions

### 1. Keep The Headline

The draft keeps the compact sentence recommended by v0.2:

```text
For fixed observer, task, horizon, and resource boundary,
future capability need not factor through observer-visible state.
```

This is now the governing sentence.

### 2. Move From Defense To Interface

The action report did the defensive work:

- do not overclaim;
- do not leave `Cap` vague;
- do not deny prior art its full state;
- do not use physics as evidence.

The new North Star draft instead states the interface:

```text
pi
Cap
K
capability equivalence
factorization up to equivalence
preservation controls
non-factorization witnesses
same-neighbor-data condition
residue ladder
```

### 3. Rename Controls

The draft avoids the confusing experimental phrasing:

```text
positive controls
negative controls
```

and uses:

```text
preservation controls
non-factorization witnesses
```

This keeps the meaning but reduces ambiguity.

### 4. Add A Residue Ladder

The decision gate is now more granular:

1. formal residue;
2. translation residue;
3. heuristic residue only;
4. delete or demote as redundant.

This prevents "absorbed vocabulary" from sounding like a total failure. A
translation layer can still be useful.

### 5. Make State Enrichment Explicit

The new draft explicitly says:

```text
yes, enrich the projection if that restores factorization
```

This keeps the candidate honest. The audit asks which projection is sufficient,
not whether the first projection deserves protection.

### 6. Keep Physics, But Quarantine It

Physics remains in the Candidate North Star, but only under:

```text
Known physics
Safe bridge
Not claimed
Test route
Falsifier / likely absorber
```

The draft preserves sections for gravity, cosmology/dark energy, dark matter,
black holes, quantum theory, Standard Model structure, time, and emergence, but
marks them as analogy, stress test, or likely absorbed comparison surface.

## Important Substantive Upgrades

The new draft is stronger because it can say both:

```text
Cap does not factor through pi
```

and:

```text
Cap does factor through pi
```

That is the difference between a testable schema and a slogan.

It also makes prior-art absorption part of the workflow:

```text
absorbed by POMDPs
absorbed by control theory
absorbed by resource theory
absorbed by provenance/access control
absorbed by sheaf obstruction
```

are legitimate audit outputs.

## Open Risks

The candidate still has three live risks.

1. `Cap` may reduce cleanly to known structures in every important case.
2. The same-neighbor-data condition may be hard to satisfy for real examples.
3. Physics-facing language may still be more inspiring than probative.

These risks are not hidden in the draft. They are part of the collapse
condition.

## Recommended Next Moves

1. Build `Capability Factorization Audit Template v0.1`.
2. Build three preservation controls.
3. Build three finite non-factorization witnesses.
4. Run same-neighbor-data absorption against POMDPs, control theory, resource
   theory, provenance/access control, and sheaf obstruction.
5. Decide whether the result is formal residue, translation residue, heuristic
   residue, or redundancy.

## Bottom Line

The new Candidate North Star is no longer mainly:

```text
the world has hidden capability fibers
```

It is now:

```text
test whether a declared projection preserves a declared future capability
object, under declared equivalence, after prior art receives its legitimate
state
```

That is much harder to misuse, and much easier to kill cleanly.
