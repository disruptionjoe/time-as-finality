# Dynamical Phase-Bearing Records

## Abstract

T16 addresses the main weakness left by T13-T15. Earlier signed-readout labs
assigned `+1` and `-1` weights by hand. T16 derives signed weights from local
dynamics.

In an elementary cellular automaton, flip one seed bit and compare the
terminal state with the unperturbed terminal state. A generated trace cell
receives weight `+1` when it changes from `0` to `1`, and `-1` when it changes
from `1` to `0`.

The result is positive in a bounded toy setting. T16 finds dynamically
generated traces where identical phase-blind finality profiles have different
signed readouts, including exact cancellation.

## 1. Motivation

T13 showed that phase-blind finality profiles do not determine signed readout.
T14 integrated that result with coupling, inherited expression, proof
validation, and consensus confidence. T15 showed that the integrated result
survives a generated family.

The remaining weakness was direct:

> The signed weights were assigned manually.

T16 asks whether sign-like readout can arise from generated record traces.

## 2. Model

Use the T9 elementary cellular automaton substrate. For a rule, initial state,
seed index, and layer depth:

1. run the baseline state forward;
2. flip the seed bit and run the counterfactual state forward;
3. compare terminal states cell by cell;
4. create a record for every changed terminal cell;
5. assign direction-derived weight:

```text
0 -> 1 gives +1
1 -> 0 gives -1
```

The finality profile remains:

```text
(support, holder redundancy, branch support, reversal count)
```

It does not include sign.

## 3. Sweep

T16 evaluates:

```text
256 rules * 3 layer depths * 16 width-4 initial states * 4 seed positions
= 49,152 cases
```

Results:

| Metric | Value |
| --- | ---: |
| nonempty trace cases | `39,488` |
| mixed-sign trace fraction | `0.2504` |
| zero-readout fraction | `0.1823` |
| phase-blind profile count | `5` |
| profiles with multiple readouts | `4` |
| multiple-readout profile fraction | `0.8000` |

## 4. Separation Witness

T16 finds two generated traces with the same profile:

```text
(2, 2, 1, 2)
```

The first is:

```text
rule 2, initial 0001, seed 2
weights: +1, -1
readout: 0.0
```

The second is:

```text
rule 3, initial 0000, seed 0
weights: -1, -1
readout: 4.0
```

The profile is identical. The readout differs. Therefore the T13 separation
survives with dynamically derived signed records.

## 5. Cancellation Trajectory

T16 also finds a generated trajectory:

```text
rule 6, initial 0001, seed 1
```

Readouts across layers:

```text
0.0, 1.0, 0.0, 1.0, 0.0, 1.0
```

The zero-readout layers come from mixed-sign generated traces:

```text
+1, -1
```

This is the finite local-dynamics analogue of the T13 hand-authored
cancellation pattern.

## 6. Claim Verdict

T16 strengthens [C3](claims/C3-signed-readout-separation.md): signed readout
separation is no longer only a hand-weighted record-graph result.

It also strengthens the T9-to-T13 bridge: local dynamics can generate
direction-sensitive traces while the finality profile remains count-based and
phase-blind.

Limits:

- The substrate is a finite binary cellular automaton.
- The sign rule is simple direction of bit change.
- No complex phase, Hilbert space, Born rule, or physical interference
  experiment is derived.
- This does not replace quantum mechanics.

## 7. Next Work

The next step is to move from `+1/-1` direction signs to richer phase classes
or complex amplitudes generated by a local reversible substrate.
