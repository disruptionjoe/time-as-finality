## Operating Frame

The branch map uses the two-track discipline now local to the repo:

```text
Track 1 = North Star pursuit.
Track 2 = subordinate branch work that reports up.
```

The North Star is not demoted by difficulty. Only actual falsification demotes
it. A finishable theorem, audit, or byproduct can be valuable, but it must feed
the North Star instead of replacing it.

## Council Read

### Orthodox Professor

The promising branches are the ones that do not disappear after standard state
completion. That removes many attractive but absorbed lanes: weak-measurement
interpretation, generic consensus analogy, copy-law conservation, authority
state, full-provenance detector packets without data, and C(R) packets without
full competency/resource/permission/provenance matching.

Best branches:

- source-law reassessment beyond the paused finite-generator route;
- cross-domain shadow protection, if it can avoid identity-by-construction;
- sheaf/descent and access/provenance restriction, because those are closest to
  known mathematics but still organize the repo's real finite witnesses.

### Heterodox-Rigorous Professor

The structure is secretly resting on one unresolved choice: what counts as a
finality-native law rather than a screen-tuned target import. Branches that
avoid this question are likely low-level exploit. The live branch is not "make
more gates." It is "find a source law, or prove that the available classes do
not supply one."

Best branches:

- Track 1 source-law search with explicit naturality and hostile controls;
- protocol-stack ablation, because it tests whether isolated S1 failure was
  caused by removing necessary layers or whether the stack is only explanatory
  retrofit;
- shadow-protection theorem, because it might abstract the law burden without
  declaring a particular finite generator.

### Commercial Scientist

The most promising branches are the ones with cheap decisive tests. The best
near-term tests are not another broad synthesis. They are packet screens,
ablation suites, and single-object projection tests with declared absorbers.

Best branches:

- source-law candidate router, but only with genuinely new candidates;
- associated-sheaf / descent witness upgraded from toy fixture to an
  open-system or SBS-style system-environment simulation;
- APRD / reconstruction-boundary projection test, because one small object can
  reproduce or fail to reproduce T19, T66, and T51/T58.

### Philosopher Of Science

The repo is strongest when it can name what kills a branch. The branch map
should not confuse "open" with "promising." C(R) is open but currently lacks a
real packet. Detector provenance is operationally important but blocked until
data exists. Computational finality is mostly demoted to boundary-mode
accounting.

Best branches:

- Track 1 source law, with falsifiers stated before the candidate is chosen;
- reconstruction-boundary / accessible-independent-witness theorem family,
  because it has clear projection tests and clear demotion paths;
- protocol-stack ablation, because every layer must earn itself by predicted
  collapse under removal.

Best branches:

Best branch:

## The Promising Branches, Plainly Ranked

### 1. Track 1 source-law reassessment

This is still the main branch because it is the only branch aimed directly at
the larger model:

```text
local record dynamics
  -> finality-native source law
  -> bounded capability / shadow preservation
  -> continuum or physics-facing tests
```

The finite-generator subroute is paused, not dead. The next work must either
find a genuinely new law family or name a principled falsifier for that law
class. Re-running spent receipt-product, record-window, content-attention, or
Lorentzian-reference shapes would be local-minimum behavior.

### 2. Shadow-protection plus descent/reconstruction boundary

The most coherent feeder branch is a combined mathematical branch:

```text
APRD / reconstruction boundary
  + sheafification / effective descent
  + typed capability non-factorization
```

This branch is promising because it can become a theorem family rather than a
single analogy. It also has concrete small tests: APRD projection tests,
associated-sheaf open-system tests, and cross-domain shadow-protection tests.

### 3. Observerse protocol-stack ablation

This is the riskiest but most interesting new branch. It takes the negative
S1 results seriously by asking whether isolated S1 was the wrong unit. The only
honest way to run it is ablation: predeclare the stack, remove layers, and see
whether each removal causes the predicted collapse.

If the stack works, it could reframe the source-law search. If it fails, it
prevents the repo from hiding behind a broad observerse story.

## Branches To Protect But Not Lead With

- Quantum access / monogamy / secret sharing: useful scaffold and hard test
  source, not a finality-core discovery by itself.
- Reconstruction debt / typed loss: strong integration vocabulary and possible
  audit calculus, but prior-art pressure is still heavy.
- Detector provenance / proof-carrying finality: operationally important and
  execution-ready, but blocked until real data-bearing packets exist.
- C(R): important but currently blocked. It should not absorb attention until a
  source law or real noncompletion packet appears.

## Branches Not Promising As Lead Branches Right Now

- Generic consensus-finality analogy: too absorbable unless it becomes a
  protocol-stack ablation or event-DAG provenance test.
- Copy-law conservation / single-use key: absorbed by consumable resource
  monotones.
- BFT threshold / monogamy numerology: shut as stated.
- Records-as-rows native-attention geometry: negative as a native geometry
  generator.
- Weak measurement as near-term discriminator: dormant until a concrete
  platform supplies a predeclared independent axis.
- Computational finality as physical arrow: demoted to E2 boundary mode unless
  new assumptions appear.
- Dark matter / dark energy stress language: not a lead branch without typed
  physical capability packets.

## Council Convergence

The council converges on this branch structure:

```text
Lead branch:
  source-law reassessment beyond the paused finite-generator route

Mathematical feeder branch:
  shadow protection through APRD / descent / reconstruction-boundary tests

High-upside speculative feeder:
  observerse protocol-stack minimality by ablation

Operational support branches:
  quantum access, detector packets, proof-carrying judgments, typed loss

Parked or absorbed branches:
  C(R) without a real packet, generic consensus, copy-law, weak-measurement
  generic routes, computational-arrow routes, and records-as-rows geometry
```

The repo is not short on branches. It is short on branches that bring their own
law, reflector, invariant, or ablation falsifier. That should be the admission
rule for the next wave.
