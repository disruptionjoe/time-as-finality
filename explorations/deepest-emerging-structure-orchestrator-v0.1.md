# Deepest Emerging Structure Orchestrator v0.1

## Executive Verdict

The four independent investigations converge on this hierarchy:

```text
typed forgetting / typed loss
  -> obstruction accounting under projection
  -> access and provenance limits
  -> bounded observer reconstruction
```

This hierarchy is **supported by evidence**, but should not yet be treated as
the final organizing structure for the entire repo.

The deepest current structure is not "access" alone and not "obstruction"
alone. The strongest candidate is:

```text
typed finite restriction systems with lossy morphisms,
where admissible obstruction attribution requires explicitly named forgotten
structure.
```

Access/provenance is the physics-facing and observer-facing manifestation of
that structure. Obstruction displacement is the dynamic behavior of that
structure under projection and transport. The independent mathematical core is
the finite typed restriction/morphism theory, not Time as Finality language.

## 1. Which Frame Explains the Most Tests?

**Typed forgetting plus obstruction accounting** explains the most tests.

It covers:

- T26 restriction systems: finite local/global obstruction carrier.
- T29-T33 PO1: admissible attribution requires named forgotten structure.
- T34 chained projection: endpoint obstruction can emerge, propagate, or be
  absorbed.
- T37 typed transport: same endpoints can have different PO1 verdicts by path
  because paths accumulate different `forgotten_structure`.
- T39 CSP reframing: obstruction mechanism is known parity CSP; typed source
  and typed forgetting are the added layer.
- T40 holonic emergence: cross-level obstruction attribution still requires
  named forgotten dimensions.
- T41 category: morphisms compose, but PO1 is not a Boolean functor.
- T58/T63/T65: contextual/holonomy obstruction can be viewed as an obstruction
  whose location depends on coefficient choice, cover, and causal boundary.
- T19/T64/T66: access and provenance are concrete forms of lost or unavailable
  structure.

Access explains the Q1/first-person thread well, but typed forgetting explains
why access/provenance matters inside the broader projection framework.

## 2. Which Frame Is Most Mathematically Novel?

**Typed forgetting** is the most mathematically novel frame.

The repo should be honest that the base obstruction is often not novel:

```text
finite gluing obstruction = signed-graph / parity CSP
```

The novel layer is the typed attribution machinery:

```text
source system
target system
definable morphism
global satisfiability resource drop
named forgotten structure
preserved structure
path-dependent accumulated loss
non-functorial obstruction admissibility
```

That is more distinctive than access-boundary language and more precise than
general obstruction conservation.

Honest abstract:

## 4. Which Frame Has the Highest Long-Term Upside?

**Typed forgetting -> obstruction accounting** has the highest upside.

Reasons:

- It is not tied to one domain.
- It already has hostile non-physics examples.
- It can absorb access/provenance as a special case.
- It can connect to category theory, CSP, finite sheaf methods, and transport
  networks.
- It gives a principled way to say when a no-go result is inherited, created,
  repaired, or merely under-specified.

The physics-facing upside remains in the access/provenance branch, especially
if detector provenance or causal-boundary data can be made operational. But
that branch is not yet predictive.

## 5. Which Frame Appears to Contain the Deepest Insight?

Deepest insight:

```text
Obstruction is not meaningful without an attribution account.
```

The repo is repeatedly learning that seeing an obstruction is not enough. One
must also know:

- what richer structure had a solution;
- what projection or restriction was applied;
- what was preserved;
- what was forgotten;
- whether the forgotten structure is exactly what resolved the obstruction;
- whether later maps restore it, absorb it, or move it elsewhere.

This is deeper than access alone. Access asks "what can this observer reach?"
Typed forgetting asks "what structure was removed, and is that removal the
reason the target became obstructed?"

## Hierarchy Test

Proposed hierarchy:

```text
Typed Forgetting
    ->
Obstruction Displacement
    ->
Access Limits
    ->
Bounded Observer Reconstruction
```

Verdict: **Supported by evidence**, with one correction.

Better hierarchy:

```text
Typed loss / forgotten-structure attribution
    ->
Typed obstruction accounting under projection
    ->
Access, provenance, and threshold limits
    ->
Bounded observer reconstruction / measurement finality cases
```

Why this correction matters:

- "Obstruction displacement" is not literal conservation. T34 absorbed
  obstruction shows that obstruction can disappear when structure is restored
  or represented by a later map.
- "Access limits" are not always lower-level than obstruction: in T19 the
  access boundary is the obstruction.
- "Typed forgetting" is broader than access: access loss, provenance collapse,
  coefficient change, path-specific loss, and projection-created obstruction
  are all special cases.

## Candidate Theorem Stack

### 1. Loss Kernel Formalization

Define a first-class object:

```text
LossKernel(f) = typed structure removed by morphism f
```

with identity and composition:

```text
LossKernel(id) = empty
LossKernel(g o f) = compose_loss(LossKernel(f), LossKernel(g))
```

This is the immediate missing step.

### 2. Typed Forgetting Attribution Theorem

For a definable projection from a globally satisfiable source to an obstructed
target, obstruction attribution is admissible only when a non-empty typed loss
kernel names structure that can explain the satisfiability resource drop.

### 3. Path-Dependent Attribution Theorem

In a typed transport network, two paths with the same endpoints can have
different obstruction-attribution verdicts exactly when their accumulated loss
kernels differ in attribution-relevant structure.

### 4. Calibration Insufficiency Corollary

In access/provenance record systems, improving local quality or calibration
cannot change finality verdicts caused by inaccessible records, duplicate
provenance classes, or unspecified threshold policy.

### 5. Reconstruction Boundary Corollary

Observer-local reconstruction is canonical only when access, identity maps,
overlap witnesses, provenance, and compatibility/descent conditions are
available. Local validity does not imply global reconstructability.
