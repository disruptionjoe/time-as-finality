# North Star v0.8 Research Improvements

## Status

This file synthesizes the 62-persona steelman and Hegelian pass into concrete
research improvements. It does not update claim status, roadmaps, test ledgers,
or the claim ledger.

The implemented changes are scaffolds:

- one reusable audit template;
- three open formalization or theorem targets.

They are meant to make future North Star work easier to reject, repair, or
type.

## Main Diagnosis

The 62-persona panel converged on one practical diagnosis:

```text
The North Star should not try to name the final geometry first.
It should build a Shadow Atlas first.
```

A Shadow Atlas is a collection of domain charts. Each chart states:

- source structure;
- observer shadow;
- capability object;
- native comparison;
- absorber completion;
- failure mode;
- minimal repair;
- demotion label.

Only after multiple charts share transition rules should the repo try to name
one global geometry.

## Improvement 1 - Add A Shadow Audit Template

Problem:

North Star probes can drift into narrative unless each one names its source,
shadow, capability object, native comparison, absorber pass, and kill
condition.

Implemented:

```text
workflows/templates/north-star-shadow-audit.template.md
```

Use:

Fill this before promoting any v0.8 vision sentence into a research artifact.
It forces:

- known theory or mature field;
- source and shadow;
- observer/access profile;
- capability object;
- absorber passes;
- geometry vocabulary status;
- verdict and demotion condition.

## Improvement 2 - Start The Cross-Domain Theorem Target

Problem:

The maximal candidate theorem is attractive but still rhetorical unless it can
be bounded to two or more mature domains.

Implemented:

```text
open-problems/cross-domain-shadow-protection-theorem.md
```

Use:

Run one computational/formal chart and one physics-facing chart through the
same proof spine:

```text
visible fiber -> capability spread -> native absorber -> minimal enrichment
```

If the proof spine does not transfer, the North Star remains an atlas problem,
not a theorem target.

## Improvement 3 - Start The Finite-To-Smooth Bridge Gate

Problem:

Finite TaF witnesses cannot carry smooth geometry language unless a bridge is
declared. This was one of the strongest repeated warnings from the panel.

Implemented:

```text
open-problems/finite-to-smooth-shadow-bridge.md
```

Use:

Any future attempt to connect finite D1, TTN, LossKernel, detector provenance,
or record-graph work to smooth geometry should name a bridge type:

```text
discretization map
inverse or direct limit
nerve / simplicial realization
sheafification
coarse-graining functor
continuum scaling limit
measurable-field construction
finite cover approximation
category of models with comparison functor
```

No bridge means the geometry language stays vision-only or source-vocabulary.

## Improvement 4 - Start The Observer-Shadow Category Target

Problem:

Many personas wanted categorical structure, while the minimalist/skeptic
cluster warned that set-level fiber constancy may be enough. This should be
tested, not assumed.

Implemented:

```text
open-problems/observer-shadow-category.md
```

Use:

Try to express two existing finite families with one object/morphism schema.
If composition does not preserve shadow-protection verdicts, record the
obstruction and keep the atlas approach.

## Research Operating Rule Added By This Pass

Use this rule for future North Star work:

```text
No global geometry before charts.
No chart without source, shadow, capability, native comparison, absorber, and
kill condition.
No smooth claim without a bridge.
No categorical claim without a composition test.
```
