## Result

### Window-Scoped Motif Taxonomy

The `T55`-`T92` span compresses cleanly into **three macro-motifs** built from
the requested five motif families.

| Macro-motif | Included motif families | Core mechanism | Representative tests |
| --- | --- | --- | --- |
| Witness-carrying records | `record`, `provenance` | Same visible payload can support different verdicts because admissible witness ancestry, independence class, or raw-log contract differs. | `T55B`, `T66-T68`, `T70`, `T72`, `T74-T79`, `T81`, `T83`, `T85-T87` |
| Bounded visibility gaps | `gap`, `access` | Ambient or global content exists, but local cover/observer/access window cannot audit it without extra overlap, coefficient, support, or witness access. | `T56-T60`, `T62-T65`, `T69`, `T73`, `T88-T92` |
| Reconciled settlement | `reconciliation` | Local provisional states, branch choices, or finality traces require later authority, rollback, export, or resource accounting before becoming stable. | `T55`, `T61`, `T71`, `T80`, `T82`, `T84` |

### The Five Requested Motifs In Their Leanest Form

#### 1. Record motif

The stable record result in this window is not "more records imply more truth."
It is:

```text
what counts is the witness packet carried by a record,
not the payload alone
```

This is visible in:

- `T55B`: same record basis, different propagation history;
- `T72/T78/T87`: raw-log and pre-registered contract load-bearing;
- `T79`: dashboard summaries are not enough;
- `T89/T92`: a proposition can be ambiently true while locally unauditable.

#### 2. Gap motif

The gap branch compresses to:

```text
ambient truth minus locally auditable truth
```

This is the shared `H0` shape across:

- `T56-T58`: apparent-finality gap machinery;
- `T57`: restriction discipline for the gap object;
- `T89/T92`: accessible witness gap for proposition-domain witnesses.

`T59`, `T63`, `T65`, `T69`, and `T73` act as guardrails: if coefficient,
support, loss-morphism, or holonomy data are underdeclared, the gap language
overstates what the tests actually prove.

#### 3. Provenance motif

The provenance branch is the strongest repeated mechanism in the window.

Its compressive form is:

```text
same passive statistics or same endpoint state
does not imply same independence or admissibility class
```

This is the common core of:

- `T66-T68`: calibration and passive correlation are insufficient; intervention
  or signed ancestry is needed;
- `T70`, `T74-T79`, `T81`, `T83`, `T85-T87`: detector evidence survives only as
  raw-log provenance discipline with predeclared partitions and contracts.

#### 4. Access motif

The access branch says:

```text
record existence != observer-usable finality
```

This is the shared predicate behind:

- `T60`: closure is reachable, but closure is not the same as internal audit;
- `T62/T64`: decoherence or detector redundancy can exist outside the observer
  access window or independence filter;
- `T89/T92`: decisive witness records may remain outside the local causal region;
- `T90/T91`: weak-measurement routes remain dormant until an independently
  accessible axis changes the verdict.

#### 5. Reconciliation motif

The reconciliation branch contributes one consistent systems lesson:

```text
provisional local commitment and stable authoritative settlement are distinct
states, and rollback/export cost must be explicit
```

This is instantiated by:

- `T55` and `T61`: conflict data, rollback, and explicit authority;
- `T71`: tradeoff regimes between progress and finality surfaces;
- `T80`, `T82`, `T84`: reversibility and cyclic behavior weaken any naive
  monotone-finality reading.

### The Smallest Honest Compression

If the window is read as one complex-systems program, the strongest compression is:

```text
bounded observers hold witness-bearing records;
access limits create local/global audit gaps;
reconciliation decides which provisional records survive as usable settlement.
```

That is smaller and cleaner than carrying separate storylines for:

- sheaf gaps;
- detector redundancy;
- provenance partitions;
- weak-measurement hopes;
- MMO reconciliation;
- consensus tradeoffs;
- reversible definalization.

Those are not separate branches at the motif level. They are stress tests of
the same three macro-mechanisms.

## What This Compression Clarifies

1. The detector corridor `T66-T87` is mostly one motif family, not many:
   provenance-qualified record admissibility under bounded access.
2. The T19/T58/T89/T92 line belongs to the same bounded-visibility family even
   when the local object changes from order pairs to unary propositions.
3. The reconciliation corridor is where attractive time-direction language gets
   demoted into authority, rollback, export, and resource conditions.
4. The `T55-T92` window does not primarily support a new ontology. It supports
   structural compression around witness-bearing records, bounded visibility,
   and settlement discipline.

membership. That matters if this map is later turned into a registry or an
automated classifier.
