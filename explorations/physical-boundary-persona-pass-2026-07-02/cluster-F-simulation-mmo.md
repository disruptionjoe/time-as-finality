## 16. Dynamical Systems Expert (Elena Voss)

### U1 — meaning of "physical rather than declared"
- **Steelman:** A boundary is physical when it is a *dynamically forced loss of
  past-determinacy under the flow* — the region's state lies on a merged
  trajectory bundle whose pre-images cannot be resolved from R-supported
  observables at any budget. Not "we chose not to look" but "the flow has
  collapsed distinct pasts into one shadow orbit."
- **Support:** Contracting/non-invertible maps genuinely destroy coordinates;
  Lyapunov contraction along stable manifolds is coordinate-free and
  agent-independent. This is what "unrecoverable" should mean.
- **Hidden assumptions:** A well-defined flow with contracting directions
  exists; the model is not globally invertible (else no true loss); "shadowing"
  approximations don't secretly restore the past.
- **Vindicate:** A finite fixture where R-restricted state sits on a genuine
  non-injective image (two distinct global pre-images, provably equal marginals
  under all R-interventions) AND this non-injectivity is a property of the
  update rule, not of a stipulated trace.
- **Kill:** The "loss" is a partial trace keyed on a declared retained set —
  i.e., the global map is unitary/invertible and only bookkeeping hides the
  pre-image (the T411 wound).
- **Confidence:** 4

## 57. Game-Mechanism Designer

### U1
- **Steelman:** A move is *physically* un-take-back-able when the **rollback
  buffer has been overwritten** — the authoritative state has committed and the
  information needed to reconstruct the prior state no longer exists in the

  information is gone*. This is a clean, operational, agent-independent test:
  can any actor, with the whole machine, reconstruct the pre-state? If the undo
  buffer is overwritten, no.
- **Support:** Every real commit/rollback system distinguishes "flagged
  committed" from "log truncated." The latter is checkable and hard: absence of a
  pre-image in the entire machine state.
- **Hidden assumptions:** The "whole machine" is finite and the buffer really is
  overwritten (data destroyed), not merely marked stale.
- **Vindicate:** Fixture where the pre-state is provably unrecoverable from the

  destruction, e.g., a many-to-one update with no retained ancilla anywhere.
- **Kill (finite closed model):** In a closed reversible fixture the "buffer" is
  never truly overwritten — the pre-image is retained in the global unitary
  (co-present tier-1 carriers, T411). "Overwritten" then means "we declared this
  register no longer part of R," which is exactly *declared traced-out*. In a
  closed finite model you cannot distinguish "buffer overwritten" from "declared
  traced-out."
- **Confidence:** 4

**Phase 2**

  from the whole machine).

  overwritten — the "old value" is recoverable from global state because the
  update is a bijection. Overwriting is a *local* phenomenon relative to a chosen
  scope; globally the pre-image persists (this is precisely the T411 kill: only
  tier-2 departs, tier-1 retained). So "buffer overwritten" collapses to "scope R

  update is *globally many-to-one* — undo information destroyed from the WHOLE
