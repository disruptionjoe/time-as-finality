## Per-Intuition Review

### A. Expansion Creates Maintenance Pressure

| Requested item | Review |
| --- | --- |
| Strongest version | Expanding a system's accessible option/state/resource space increases the set of possible configurations faster than maintenance capacity, so only configurations inside a constrained viability kernel persist. |
| Weakest point | This is very close to known selection, control, and viability language. "More possibilities" alone does not create selection unless maintenance, disturbance, and viability criteria are specified. |
| Closest prior art | Viability theory, evolutionary dynamics, open-system thermodynamics, control theory, ecological resilience, niche construction. |
| Potential mathematical object | A controlled transition system with possibility set `X`, disturbance set `D`, repair actions `A`, maintenance budget `B`, and viability set `V`. |
| Potential executable model | Expand the number of candidate structures while holding repair budget fixed; measure which structures remain inside `V` over horizon `h`. Include a control where expansion adds possibilities but no new viable structure. |
| Falsification route | If persistence is fully predicted by standard stability/control/energy variables and no observer/task access variable splits cases, TaF adds no residue. |
| Disposition | Preserve as a framing, formalize only through a viability-kernel or finite transition model. Do not promote. |

### B. Emergence As A Partial Solution To Maintenance Problems

| Requested item | Review |
| --- | --- |
| Strongest version | Emergent structures matter when they lower the marginal cost of maintaining, verifying, coordinating, or reproducing downstream structure. Language, trust, institutions, protocols, and scientific norms are maintenance-cost reducers. |
| Weakest point | Many examples are institutional economics, cybernetics, or information processing in ordinary language. "Reduces cost" must be measured, not inferred from usefulness. |
| Closest prior art | Ostrom commons governance, transaction-cost economics, cybernetics, database operations, error-correcting codes, social epistemology, cultural evolution. |
| Potential mathematical object | A cost-transforming macrostructure: `E` maps task family `T` from maintenance cost `C_before(t)` to `C_after(t)` while preserving admissibility. |
| Potential executable model | Compare two coordination systems with matched agents, noise, and goals: one without shared protocol and one with language/trust/consensus convention. Score audit cost, enforcement cost, and downstream operation count. |
| Falsification route | If the "emergent" object only moves cost to an uncounted subsystem, or if all cost reduction is ordinary compression/coordination already captured by known models, demote. |
| Disposition | Formalize as active exploration if tied to a cost ledger and negative controls. Otherwise preserve as analogy. |

### C. Structure That Begets Structure

| Requested item | Review |
| --- | --- |
| Strongest version | The important object is not a stable structure but a maintained platform that grants downstream operation rights for further structures. |
| Weakest point | "Begets" can become vague importance language. A platform must be defined by measurable affordances or enabled operations. |
| Closest prior art | Niche construction, causal emergence, scaffolding/extended cognition, institutional economics, constructor theory, autocatalytic/autopoietic systems. |
| Potential mathematical object | `Platform(S, O, h, R)` meaning structure `S`, for observer/task class `O`, over horizon `h`, enables downstream operation set `R` while meeting maintenance constraints. |
| Potential executable model | Use T114-style twins: two structures matched on geometry, dynamics, maintenance, and persistence, but only one maintains accessible witnesses or permissions that allow downstream operations. |
| Falsification route | If platform status is fully determined by persistence time or operation count alone, with no record/access/maintenance residue, the idea reduces to ordinary utility or reachability. |
| Disposition | Formalize. This is the strongest surviving intuition, but only as platform-capacity accounting, not as a North Star promotion. |

### D. Maintenance Loop

| Requested item | Review |
| --- | --- |
| Strongest version | Maintenance enables persistence; persistence creates a window for higher-order structure; higher-order structure may lower or reproduce maintenance capacity; the loop can become self-stabilizing. |
| Weakest point | The loop is circular unless one arrow is independently measured. "Maintenance causes persistence causes maintenance" is not explanatory without a perturbation or intervention. |
| Closest prior art | Autopoiesis, organizational closure, Ashby ultrastability, homeostasis/allostasis, cybernetics, active inference, closure of constraints. |
| Potential mathematical object | A feedback system where maintenance capacity `M_t`, structure state `S_t`, and downstream platform capacity `P_t` update by explicit transition rules. |
| Potential executable model | Ablate one loop component: remove repair, remove memory, remove feedback, or remove downstream platform support; test which loop collapses and which survives. |
| Falsification route | If no intervention on maintenance capacity changes persistence or downstream platform status, the loop is decorative. If the loop only restates the transition rule, demote. |
| Disposition | Preserve, but formalize only after A-C have finite models. Do not use as a primitive. |

### E. Consciousness As Maintenance

| Requested item | Review |
| --- | --- |
| Strongest version | Consciousness-facing work may be reframed as maintenance of a self-world model through sensory, memory, attention, body-ownership, social, language, self-reference, and error-correction protocols. |
| Weakest point | This is extremely absorbable by predictive processing, active inference, cognitive control, global workspace, self-model, and extended/scaffolded cognition. It also risks smuggling first-person claims into a maintenance metaphor. |
| Closest prior art | Active inference/free energy, predictive processing, global workspace, self-model theory, extended/scaffolded cognition, autopoiesis, cybernetics. |
| Potential mathematical object | A layered self-world maintenance stack with state estimates, error signals, memory refresh, attention allocation, body-boundary constraints, and social/language reconciliation. |
| Potential executable model | Simulate an agent maintaining a self/world latent model under memory corruption and social correction; compare task survival, identity continuity, and reconstruction debt under protocol ablations. |
| Falsification route | If all explanatory work is done by predictive-processing or control variables, with no distinct record-finality/access variable, keep this as literature mapping only. |
| Disposition | Preserve as speculative exploration. Do not formalize in core TaF until non-conscious record systems instantiate the same maintenance-stack object. |

### F. Expansion -> Selection -> Emergence

| Requested item | Review |
| --- | --- |
| Strongest version | Simple generative rules can expand the possibility space; limited maintenance capacity selects persistent configurations; some selected configurations become platforms for further emergence. |
| Weakest point | This is nearly a generic complexity/evolution story. It needs a nontrivial order parameter: maintenance-constrained platform capacity, not "complexity appears." |
| Closest prior art | Evolutionary dynamics, artificial life, cellular automata, complex adaptive systems, self-organization, niche construction, causal emergence. |
| Potential mathematical object | A generative transition system with expanding candidate set, resource-bounded maintenance, selection filter, and downstream platform-affordance score. |
| Potential executable model | Cellular automaton or graph rewrite system: generate structures, impose finite repair/audit capacity, score which structures persist and enable future transformations. |
| Falsification route | If emergence frequency tracks only known CA/dynamical variables and maintenance/access variables add no predictive split, demote to complexity vocabulary. |
| Disposition | Preserve as executable-model candidate. Formalize only with a declared order parameter and negative controls. |

### G. Conservation Of Consequences

| Requested item | Review |
| --- | --- |
| Strongest version | Information can cease to be available as a record while leaving operational residue as habit, commitment, constraint, selection landscape, authority boundary, or reconstruction debt. |
| Weakest point | "Conservation" is probably false or misleading. Consequences are transformed, amplified, erased, or made inaccessible; they are not generally conserved. |
| Closest prior art | Abstract interpretation, provenance/why-not provenance, causal histories, path dependence, hysteresis, institutional commitments, habit formation, reconstruction debt, LossKernel T107/T108. |
| Potential mathematical object | A source-fiber effect map: lost source witness `w` induces target-side operation boundary, obligation, or stable constraint if target judgments remain source-lift dependent. |
| Potential executable model | Reuse P97 axis-crossing shape: phase/history loss changes access admissibility; orientation loss changes merge authority; decorative-label loss has no effect. |
| Falsification route | If every alleged consequence is either directly present as ordinary provenance or has no measurable effect on later operations, abandon conservation language. |
| Disposition | Preserve only after renaming. Use "relocation of operational consequence" or "residual obligation," not conservation. |

### H. Emergence As Constraint Transformation

| Requested item | Review |
| --- | --- |
| Strongest version | Emergence is not creation from nothing; it is transformation of constraints across representations or scales, where some micro degrees of freedom disappear while macro operation rights, obligations, or impossibility surfaces appear. |
| Weakest point | This may be exactly abstract interpretation, coarse-graining, renormalization, causal emergence, or constraint closure in different vocabulary. |
| Closest prior art | Abstract interpretation, renormalization/coarse-graining, causal emergence, category-theoretic abstraction, closure of constraints, organizational closure, LossKernel/source-fiber relocation. |
| Potential mathematical object | A scale/representation morphism `f: S -> T` with preserved constraints, discarded witnesses, induced target obligations, and downstream operation rights. |
| Potential executable model | Build paired source-target systems where the same macro state arises from multiple micro states; test whether hidden micro constraints become target operation boundaries. Include absorbed-loss and decorative-loss controls. |
| Falsification route | If standard abstraction/concretization or coarse-graining fully predicts all target constraints and obligations, TaF adds only local notation. |
| Disposition | Formalize as the second-strongest candidate, but under prior-art pressure. This is likely the mathematical form of C and G. |

## Are A-H Separate Ideas?

They are not fully separate. The room synthesis compresses them into one deeper
candidate concept:

```text
Maintenance-constrained viability transfer:

An expanded possibility space creates more candidate structures than can be
maintained. Under finite maintenance, audit, and repair capacity, some
structures remain viable. A subset of viable structures preserves or transforms
constraints/witnesses in a way that grants downstream operation rights. Those
structures become platforms for future structure formation.
```

Mapping:

| Intuition | Role in the compressed concept |
| --- | --- |
| A | Source of pressure: expanded possibilities exceed maintenance capacity. |
| B | Mechanism: some emergent structures reduce maintenance or coordination cost. |
| C | Payoff: maintained structures become downstream platforms. |
| D | Feedback: platform capacity can reproduce or modify maintenance capacity. |
| E | Speculative application: self-world modeling as a maintenance stack. |
| F | Dynamical narrative: generative expansion plus finite maintenance yields selection and persistence. |
| G | Residue: lost records can survive as obligations, constraints, or operation boundaries. |
| H | Formal bridge: emergence as constraint/witness transformation across representations. |

The strongest formal core is therefore:

```text
viability under finite maintenance
  + observer/task-indexed witness access
  + downstream operation rights
```

The weakest and most absorbable parts are:

```text
generic expansion
generic selection
generic maintenance loop
consciousness framing
conservation language
```

## Minimal Executable Model

Working title:

```text
Maintenance-Platform Separation Audit
```

Required finite cases:

| Case | Purpose |
| --- | --- |
| expansion_no_pressure | Expanded options but maintenance budget scales with them; no selection pressure. |
| pressure_without_platform | Limited maintenance selects persistent structures, but none enables downstream operations. |
| maintenance_cost_reducer | A protocol lowers audit/coordination/repair cost for later tasks. |
| persistence_without_finality | Structure persists but lacks accessible witnesses for the observer/task. |
| finality_without_platform | Record is final but grants no meaningful downstream operation rights. |
| matched_standard_platform_split | Two cases match standard stability/control/entropy proxies but split on maintained witnesses or operation rights. |
| consequence_relocation | Lost record is no longer accessible but changes later operation boundary. |
| decorative_loss_control | Lost record has no task relevance and creates no debt or constraint. |

Pass condition:

```text
At least one matched-standard pair splits only on maintained witnesses,
reconstruction debt, or downstream operation rights; negative controls prevent
every persistence or loss event from counting as emergence.
```

Fail condition:

```text
All verdicts are functions of standard stability, control, entropy, reachability,
or ordinary abstraction variables. In that case this family is useful vocabulary
over existing theory, not a TaF-specific research line.
```

## Bottom-Line Judgment

The family is interesting, but most of it is not new theory. The strongest
nontrivial residue is the conjunction:

```text
maintained witness access + reconstruction debt + downstream operation rights
```

This residue is already aligned with T114, P96, and P97. The correct next move
is not claim promotion. It is a finite separation audit that tries to show a
matched-standard split:

```text
same ordinary stability / entropy / control / viability variables
different observer-finality or platform-capacity verdict
because witness maintenance or operation rights differ
```

If that split cannot be produced without post hoc labels, abandon the family as
new theory and keep it only as a useful organizing vocabulary over viability,
control, resilience, active inference, provenance, and abstraction theory.
