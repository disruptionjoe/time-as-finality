Status: primary sources checked 2026-06-22.

## Current Repo Verdict

Temporal Issuance does not currently reopen H7's physical-arrow route. After
T172, its useful role is narrower:

```text
Temporal Issuance is a candidate source-side vocabulary whose value depends
on whether its realization order and measure survive absorber tests against
ordinary causal order, stochastic thermodynamics, information theory, and TaF
record reconstruction.
```

The present note does not import Temporal Issuance as true. It asks whether
the bridge has any typed residue once neighboring theories are granted their
native state variables.

## Same-Neighbor-Data Vector

A non-null Temporal Issuance proposal must freeze this vector before comparing
TaF readouts:

```text
(causal order,
 spacetime or causal-set volume/counting data,
 observer worldlines/access regions,
 detector sampling/cadence,
 thermodynamic path and reservoir ledgers,
 information-state and compression/statistical-complexity ledgers,
 record-generation rule,
 gluing/identity data,
 gauge/label/foliation convention)
```

If two candidate source systems differ in any of these fields, the difference
belongs to the absorber until the proposal explains why that field is not
legitimate same-neighbor data.

## Consequence For T172

T172 showed that a sound record chain can reflect visible source order, but it
also showed that hidden issuance, `mu`, access loss, cadence variation, and
gluing ambiguity are not preserved by the plain TaF readout.

N13 adds the absorber interpretation:

```text
The lost source fields are not automatically new physics. They are likely
ordinary causal, thermodynamic, information-theoretic, instrumentation, or TaF
gluing fields until a same-neighbor-data split is produced.
```

This weakens the bridge. The live question is no longer "does issuance explain
time?" The live question is:

```text
Is there a non-circular, covariant, operationally typed source object whose
observer record projection preserves a capability not already absorbed by
causal order, thermodynamic accounting, information state, or record
reconstruction?
```

## Promotion Gate

Temporal Issuance can become more than translation residue only if a future
artifact supplies all of the following:

1. A typed source state space `Y_TI` whose order `<` is either explicitly
   ordinary causal order or explicitly not reducible to it.
2. A `mu` with units, operational meaning, transformation behavior, and
   comparison rule.
3. A local boundary/frontier object that is covariant or label-invariant and
   does not select a preferred global present.
4. Observer cadence/access variables that are not hiding ordinary clock,
   detector, or channel-bandwidth data.
5. A source-to-record generation map that does not define issuance through
   finality, stabilized records, or reconstruction success.
6. Positive preservation controls and hostile same-neighbor-data splits after
   causal, thermodynamic, information, and TaF gluing variables are matched.
7. A declared capability object that is not just "recover the hidden issuance
   variable."

## Demotion Rule

Demote the Temporal Issuance bridge to translation residue if:

- `<` is ordinary causal order or causal-set order with renamed vocabulary;
- `mu` is volume/counting measure, entropy production, free-energy drawdown,
  action-like bookkeeping, Shannon information, statistical complexity, or a
  record-counting score without a surviving split;
- `dR` requires a preferred global frontier;
- `kappa_i` is just observer proper time, sampling cadence, or detector
  bandwidth;
- records are needed to define issuance rather than to read it out;
- the bridge factors through existing TaF record reconstruction or gluing
  machinery; or
- the alleged residue disappears once causal, thermodynamic, information,
  access, cadence, and gluing variables are treated as legitimate native data.

## Current Strongest Surviving Use

The surviving use is methodological:

```text
Temporal Issuance can serve as a source/readout stress test for TaF. It forces
H7 to ask what finality preserves from a declared source process, rather than
asking finality alone to generate a physical arrow.
```

That is useful, but it is not a physics claim.
