# Composite Absorber Stack Progress Lanes

Status: progress-selection artifact.

This preserves the five composite absorber-stack ideas surfaced after the
all-persona synthesis and Bridge-Functor admission packet work. It does not
move claim status, public posture, North Star text, roadmap status, hard policy,
or cross-repo truth.

## Frame

The current useful question is:

```text
What does a declared stack of known absorbers explain, and does any residual
remain after the stack is granted in its strongest native form?
```

These lanes are more viable than another analogy-first restart because each
names:

- a phenomenon that previously looked mysterious;
- a stack of known absorbers that may explain it;
- a possible residual if the explanation does not fully close.

## Top Five Lanes

### 1. Bounded Retrieval Stack

Stack:

```text
observer budget
+ retained-state summary
+ retrieval workload
+ full-history absorber
+ source-checked lower-bound literature
```

Why useful:

T495 already shows that full-history visibility factors arbitrary indexed
retrieval while bounded committed summaries do not, even though those summaries
pass their native retained-task controls. This is the most concrete current
composite explanation of finality-like behavior: a committed record can be
final for native retained tasks while losing arbitrary recollection.

Best next progress:

Use the Bridge-Functor packet on a source-checked T495 successor. The output
should be a bounded-record retrieval composite explanation or review target,
not attention/quantum/physics mechanism language.

### 2. Kappa Across Absorber Genres

Stack:

```text
same invariant
+ multiple unrelated native absorbers
+ no retuning
+ native witness checks
```

Why useful:

The kappa line is the repo's best historical example of a composite residual
pattern: one object transported through several unrelated absorber genres
without per-domain retuning. It should be treated as a methodological template,
not as a license to reuse kappa everywhere.

Best next progress:

Use the kappa line as the structural model for future composite-residual
packets: same object, unrelated native absorbers, native witnesses, no
post-hoc tuning.

T499 result: the T224-T244 kappa catalogue clears this core structural template
only as method: fixed invariant, unrelated absorber genres, no retuning, native
witnesses, falsifying controls, and rank load-bearing. T465/T466 still cap the
line below prediction-language or theorem-language unless a future non-identity
target witness clears independent source-rank and predeclared-map burdens.

### 3. Authoritative Commit / Settlement Stack

Stack:

```text
local visible record
+ event provenance
+ ledger/server authority
+ rollback/reconciliation rules
+ adversarial completion
```

Why useful:

This is the most finality-native candidate. In ledgers, games, distributed
systems, and simulations, the apparent finality often lives in the interaction
between local state and authoritative settlement/provenance state.

Best next progress:

Build a tiny commit/settlement C(R) fixture with fixed local visible state,
explicit completion rights, and full log/ledger/server absorbers. The useful
output may be composite explanation rather than residual.

### 4. Competency + Resource + Permission Stack

Stack:

```text
full task-success competency profile
+ resource preorder
+ boundary/permission completion
+ provenance completion
```

Why useful:

T493-T494 strongly suggest that current C(R) readings are absorbed by full
competency-style task-success profiles. That can still be progress if C(R)-like
behavior is best explained as a composite operational profile rather than a new
primitive.

Best next progress:

Ask whether any C(R) residual survives after granting competency, resource
preorder, permission lattice, and provenance completion together.

T500 result: current T493/T494 C(R) material is absorbed as a composite
operational profile once full competency profile, resource preorder,
permission lattice, and provenance completion are all granted. Weak
single-layer readings are rejected, and only a future packet with controlled
capability spread after the full stack is admitted for review.

### 5. Typed Translation / Object-Identity Stack

Stack:

```text
typed schema interface
+ relabeling quotient
+ object-identity criterion
+ gauge/coordinate quotient
+ preservation theorem
```

Why useful:

T492 found that two gap constructions can share a typed schema without earning
object identity. This is a disciplined route back toward geometry: shared
interface explains why the analogy is meaningful, while identity and gauge
absorbers explain why promotion is blocked.

Best next progress:

Try a typed preservation-theorem packet. If it fails, record the composite
explanation: same interface, not same object.

T501 result: the T92/T113 bridge preserves the typed-gap schema interface under
relabeling and quotient discipline, but object identity remains blocked because
carrier kind, target kind, and admissibility rule differ. No direct
preservation theorem is supplied for the actual pair, so the result is a
composite explanation: same interface, not same object. A synthetic exact-object
packet is admitted only as a future review target when same carrier, same
target, same admissibility rule, direct preservation theorem, and invariant
obstruction all survive the stack.
