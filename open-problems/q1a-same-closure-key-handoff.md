# Q1A Same-Closure-Key Handoff

## Status

Internal/external reopen burden for Q1A. This is not a measurement result, not
an interpretation upgrade, and not evidence for Q1A beyond its current
`bookkeeping_only` status.

## Route

Quantum measurement / classical records.

## Why This Exists

T140 says no Q1 child branch currently has an active internal upgrade route.
For Q1A, T104 absorbed the strongest fixed-data witness into provenance-aware
Quantum Darwinism, and T105, T109, T118, and T147 closed the remaining
current-family escape hatches.

This file converts that closure into a concrete handoff. Its purpose is to
prevent more internal Q1A toy models from being mistaken for progress unless
they beat the exact closure key already earned.

## Current Strongest Claim

Q1A survives only as an admissibility rule over already formed records:

```text
Do not treat raw fragment redundancy as observer-relative classical finality
until the access cut and provenance-aware independence partition are physically
audited.
```

Within the current fixed-data family, all verdict content factors through:

```text
partition visibility + audited accessible provenance-support count
```

That is the only live closure key in the present family.

## Reopen Burden

Q1A should be revisited only if one of the following is supplied.

### 1. Same-Closure-Key Witness

A future artifact holds fixed all of the following:

1. Standard quantum-side summaries:
   - decoherence / pointer-basis evidence;
   - fragment-information summaries;
   - ordinary branch/history availability.
2. The audited provenance-aware fragment partition.
3. Partition visibility.
4. Audited accessible provenance-support count.

Then it still changes the D1 verdict through a predeclared physical dimension
that is not equivalent to:

- raw fragment count;
- relabeled branch/history variation;
- hidden partition changes;
- or an undo/reversal proxy that collapses back to support counting.

### 2. Physical Partition-Rule Derivation

A future artifact derives the admissible partition or threshold rule from
additional physical structure that the neighboring framework does not already
own and cannot import without adding new physics.

The burden is not "tell a better story about records." The burden is either a
same-closure-key verdict split or a genuinely extra physical rule.

### Minimum Predeclared Burden

Before a new Q1A witness is treated as live, it must declare:

1. The fixed standard quantum-side summaries it is holding constant.
2. The audited provenance-aware partition it shares with the neighbor.
3. The closure key values: partition visibility and accessible provenance
   support.
4. The candidate physical dimension expected to split the D1 verdict.
5. Why that dimension is not already absorbed by decoherence, Quantum
   Darwinism, consistent histories, RQM, QBism, or Everett-style bookkeeping.
6. A demotion rule stating exactly what would show the new dimension collapses
   back to the existing closure key.

If those are not declared up front, the artifact is bookkeeping-by-default.

## Null Outcomes

Treat a proposed Q1A move as null if any of the following occur:

- It changes ordinary branch/history or decoherence data while claiming to be a
  fixed-data witness.
- It reintroduces raw redundancy under a new name.
- It hides a changed provenance partition inside the witness description.
- It uses reversal or erasure language without a calibrated observable that
  survives T118.
- It treats "observer-relative" language alone as a distinction from RQM,
  QBism, or Everett.
- It claims external novelty after the neighbor is granted the same audited
  partition and access cut.

## Decision Rule

If no same-closure-key witness or physical partition-rule derivation is named,
Q1A should remain `bookkeeping_only` and should not receive more autonomous
toy-model effort.

If a candidate witness is named but fails the burden above, demote it directly
to record-accounting discipline rather than reopening Q1A by rhetoric.
