# Open Problem: Rendered Interface Assumptions

## Question

If spacetime is treated as a rendered or observer-accessible interface, which assumptions belong to that interface, and which assumptions must be preserved by any deeper-source map?

## Why This Matters

Speculative claims about deeper source structures can become rhetorical escape hatches. A no-go theorem cannot be bypassed by saying "the deeper layer is different" unless the source-to-interface map is defined clearly enough to know which assumptions are preserved, weakened, or lost.

## Required Work

- Define the source layer.
- Define the observer protocol layer.
- Define the classical record layer.
- Define what is preserved by the map.
- Define what counts as failure.

## Local Consistency Constraint

Any rendered-interface proposal must distinguish local consistency from global consensus.

Acceptable:

```text
overlapping causal regions
  -> mutually accessible records
  -> locally reconciled histories
  -> domain-relative finality
```

Not acceptable:

```text
one global renderer
  -> one global synchronized state
  -> all observers share a universal commit order
```

## Remote Observation Versus Direct Participation

A rendered-interface model must not deny ordinary astronomy. Received photons or other signals from remote galaxies are already causal records and therefore already part of an observer's record-accessible world.

The model may distinguish:

- remote observation through received signals;
- direct local participation in a region;
- inaccessible events whose records cannot reach the observer;
- later reconciliation when records enter a common causal future.

It may not claim that remote galaxies are unrendered merely because the observer cannot travel there.

## Horizon Guardrail

Event horizons may be used as stress cases for causal-access boundaries. They should not be described as transportation hubs, galaxy-to-galaxy render gates, or mechanisms for crossing into another distant rendered domain unless a separate, physics-compatible mechanism is supplied.

The useful question is:

> Which records are exterior-accessible, which records are interior-local, and which compatibility relation can an exterior observer legitimately assign?

## Mechanism Challenge

Before a rendered-interface proposal can move beyond metaphor, it must answer:

- What is computed or stabilized?
- Which systems perform the computation or record integration?
- What physical relation defines overlap between causal domains?
- What differs from standard causal-access language?
- What observation, toy model, or formal counterexample would distinguish the proposal?
