# Papers — drafts (work in progress)

Drafts, notes, deep-research/adversarial prompts, review passes, and
**previous/superseded versions**. Nothing here is a public claim. **Edit freely
and version liberally** — this is the open zone: agents may write many versions,
keep changelogs, and iterate without a gate.

Papers that have graduated for publication live in
[`../candidates/`](../candidates/); the public record lives in
[`../published/`](../published/). The three-stage lifecycle is in
[`../README.md`](../README.md).

## Conventions

- **Version in place.** Keep `-vN` files or a `CHANGELOG-*.md`; do not silently
  overwrite a prior version — supersede it and note it in the file header.
- **Grade honestly.** State the evidence grade in-text (finite-witness / computed
  / recorded / internally established), and never let wording read stronger than
  the evidence. Absorbed or downgraded results must be labeled as such, not cited
  as established.
- **Superseded stays for provenance.** Do not move superseded drafts to an
  archive; a historical reference should keep naming the file it cited. This index
  is the source of truth for what is current vs superseded.
- **One carrier per result.** If two drafts share a core, only the hardened one
  should later stage as a candidate.
