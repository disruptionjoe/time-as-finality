# papers/published/

The **public record** — only papers that have actually been **published publicly**
(arXiv, a journal, a live blog post, a talk writeup). **Empty by default.**

Joe moves a paper up here **by hand**, when and only when it actually goes public,
recording the venue / date / arXiv id / DOI / URL. No agent publishes, submits, or
promotes a paper here — this folder is the one place in the pipeline that requires
a real-world action Joe has taken.

Near-final papers that Joe has decided to publish but has not yet posted live in
the sibling [`../candidates/`](../candidates/). The full three-stage lifecycle
(`drafts/ → candidates/ → published/`) is described in [`../README.md`](../README.md).

Treat as **append-mostly**: do not silently rewrite a published artifact — publish
a revision and record it (a new versioned entry noting what changed and the new
venue/date). The published record is the repo's constitutional layer: it changes
only through an explicit, real publication event.
