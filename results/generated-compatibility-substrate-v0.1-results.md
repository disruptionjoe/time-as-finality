# T378 Results: Generated Shared-Compatibility Substrate

## Current strongest claim

A finite local compatibility generator with no source time/space/rank columns
can derive path-independent lineage ranks from a shared substrate. Once those
lineages are derived, multiple observers recover the same interval-like
invariant while disagreeing on simultaneity.

This is a stronger calibration than T377 against the explicit fixed rank-table
absorber. It is still not a claim upgrade, because the finite generator rule and
finite closure remain fixed.

## Plain-English reading

T377 said:

```text
If we hand the system a fixed u/v table, observers can render different time
coordinates while preserving the same interval-like structure.
```

T378 improves that to:

```text
The u/v-like structure does not have to be stored as source columns. It can be
recovered from a shared compatibility graph generated by local rules.
```

So the relativism is now more clearly about a shared compatibility substrate,
not about a source table that already contains the answer.

## What improved

The explicit rank-table absorber is weakened. The source rows do not store
`u_rank`, `v_rank`, time, space, metric, or interval fields. The rank-like
values are reconstructed from path counts over local compatibility-channel
edges, and reversing exported record/edge order does not change the result.

The observer result also got broader: four observers, not just two, recover the
same interval-like invariant across all 630 generated record pairs.

## What weakened

The stronger reading is still blocked:

```text
generated compatibility substrate = genuinely nonfixed relativistic substrate
```

The generator rule is fixed. The finite bounds are fixed. Because the closure is
finite, an adversary can still expand the generated object into a completed
compatibility table after the fact.
