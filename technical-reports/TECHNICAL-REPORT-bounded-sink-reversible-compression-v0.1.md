# Technical Report: Bounded-Sink Reversible Compression v0.1

## Claim Under Test

H7 says finality structure can induce an observer-relative temporal direction
when admissible transformations are monotone in D1 finality. T18 proves that
as a conditional constructor theorem. T80, T82, and T84 progressively weakened
the physical-arrow reading: raw reversible dynamics can definalize, persistent
memory needs irreversible OR memory or append-only ledger capacity, and cyclic
memory gets monotone accounting only by exporting or erasing overwritten
records.

T106 tests the remaining T84 loophole: maybe a logically reversible
compression/export scheme with a bounded entropy sink can preserve a
nontrivial finality monotone after the sink is included in the state space.

## Artifact

The model reuses the T80/T84 sequence:

```text
observed masks = (10, 11, 10, 00, 10, 11)
exported overwritten slots = (00, 00, 00, 10, 11, 10)
local ring support = (1, 3, 4, 3, 2, 3)
accounted support = (1, 3, 4, 4, 5, 7)
```

It audits three cases:

1. **Orderless counter compression.** Store only counts of exported slot
   symbols.
2. **Ordered lossless stack export.** Store overwritten slots in blank sink
   positions.
3. **Closed bounded cycle.** Include the sink and reversible unwind path in the
   state space rather than counting only the forward branch.

## Result

The result is negative for H7's physical-arrow reading.

Orderless compression is non-injective: distinct ordered histories can produce
the same count vector. Making it reversible requires extra side information,
which reintroduces an ordered history resource.

Ordered stack export is injective, but only because each overwritten slot is
stored in a blank sink position. The T80/T84 export sequence has a finite
run-length compression:

```text
00 x 3, 10 x 1, 11 x 1, 10 x 1
```

That still requires four RLE blocks. A three-block sink exhausts on export
index 5; a four-block sink works only because it is sized for this finite
sequence.

Most importantly, the closed bounded cycle kills the monotone:

```text
forward accounted support:
  0, 1, 3, 4, 4, 5, 7

full cycle after reversible unwind:
  0, 1, 3, 4, 4, 5, 7, 5, 4, 4, 3, 1, 0
```

The forward branch is monotone. The closed reversible cycle is not. More
generally, on a finite cycle any scalar nondecreasing on every edge must be
constant around that cycle. A strict finality monotone therefore cannot live in
the closed bounded reversible state space.

## Current Strongest Claim

For the T80/T84 witness, reversible compression with a bounded sink does not
rescue H7. Orderless compression is non-injective; ordered lossless export is
reversible only while consuming blank sink capacity; and when the bounded sink
is included and the cycle is closed, the apparent monotone decreases on the
reversible return path.

## What This Improved

T106 converts the T84 next-step loophole into a finite obstruction. The
load-bearing question is not whether storage is raw or compressed. It is
whether overwritten records remain reconstructable in the state space, are
erased, or are hidden in an excluded environment.

## What This Weakened Or Falsified

This further weakens H7 as a physical-arrow claim. The tested bounded
reversible sink supplies at most a forward-branch resource accounting curve,
not an autonomous finality direction after all degrees of freedom are
included.

It does not falsify T18. T18 remains a conditional constructor theorem: if
admissible transformations are D1-monotone, strict finalization induces a
direction. T106 weakens the claim that such monotonicity is physically
generated by bounded reversible observer memory.

## Falsification Condition

T106 fails if a finite bounded reversible compressor over the same T80
exported-slot family gives a strict D1-relevant monotone on the closed full
state space without erasure, excluded history, fresh blank capacity, or a
hidden side channel.

## Claim Ledger Update

Add T106 to H7:

```text
Bounded reversible compression does not rescue the physical-arrow reading.
Orderless compression is non-injective, ordered export consumes sink capacity,
and the closed bounded cycle has no nontrivial strict monotone once the sink is
included.
```

## Open Blocker

No finite closed reversible observer model has produced a strict D1-relevant
monotone after including the memory, compression sink, and return/unwind
degrees of freedom.

## Next Work

Either formalize this as a general finite-permutation obstruction for H7, or
look for an open-system thermodynamic model where H7's remaining content is
explicitly the observer-indexed coarse graining rather than a new arrow of
time.
