# Technical Report: Physical Record Deletion Fixed-Accounting Screen v0.1

## Claim Under Test

T144 left H7 with one plausible physical-arrow class:
`physical_record_deletion`. T145 tests whether the current repo has a witness
inside that class after standard absorber data are held fixed.

The fixed absorber vector is:

```text
erased bits / beta*W floor
blank-capacity change
sink or exported-history state
observer boundary
provenance / authority state
source-copy status
reversible-control access
```

## Result

No current fixture clears the H7 physical-arrow gate.

The only matched-accounting split is the T142 same-chain versus branch-spread
copy distinction. It preserves a task-specific future-operation split:
branch-spread support can survive one branch failure when same-chain support
cannot. But the deletion reverse is still ordinary one-bit blind-erasure
accounting. That is a capability/topology residue, not thermodynamic-arrow
evidence.

The other deletion-shaped cases fail because absorber data change:

- correlated uncopy versus blind reset changes reversible-control access,
  erased bits, and the free-energy floor;
- one-bit versus two-bit deletion changes erased bits, work floor, and blank
  capacity;
- hidden sink/export history changes sink state;
- access revocation is boundary access loss, not physical deletion;
- authority/provenance revocation is governance/accounting loss, not physical
  deletion.

## Current Strongest Claim

At fixed free-energy floor, blank-capacity change, sink state, observer
boundary, provenance state, source-copy status, and reversible-control access,
the current fixtures contain one real split: D1 branch topology changes future
branch-failure robustness. That does not upgrade H7 because the deletion
reverse remains ordinary erasure accounting.

## What This Improved

T145 converts the T142/T144 blocker into an executable screen. Future deletion
witnesses must now state their absorber vector and pass the narrow H7 upgrade
gate instead of relying on ambiguous "definalization" language.

## What This Weakened

T145 weakens the escape route:

```text
D1 topology differs at fixed Landauer floor
```

That can be a real future-operation distinction, but it is not a physical
arrow unless reverse deletion remains constructor-impossible after full
accounting.

## Falsification Condition

T145 fails in H7's favor if a physically typed record substrate exhibits a
`physical_record_deletion` pair with:

1. matched free-energy / erased-bit floor;
2. matched capacity and sink/export state;
3. matched observer boundary and provenance state;
4. matched source-copy and reversible-control access;
5. a task-natural future-operation split; and
6. a reverse status of `constructor_impossible_after_full_accounting`.

## Open Blocker

The repo still lacks a physically typed substrate where record deletion, rather
than access loss, support-copy loss, provenance loss, or ordinary erasure,
remains impossible for substrate-native reasons after full absorber accounting.
