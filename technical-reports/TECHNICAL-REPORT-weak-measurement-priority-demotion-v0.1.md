# Technical Report: Weak-Measurement Priority Demotion v0.1

## Claim Under Test

The repo has spent several runs asking whether T12 can still be the lead Q1
experimental route. After T83, T85, T86, T91, and T93, the more valuable
question is strategic rather than constructive: should weak measurement remain
in front of detector provenance, or has the evidence already forced a demotion?

## Result

Weak measurement should now be demoted below detector provenance in the active
roadmap.

The reason is not that detector provenance is already empirically validated. It
is not. The reason is that detector provenance still has an executable,
source-anchored, pre-registered raw-log route that survives the current null
criteria, while weak measurement has no real platform with a pre-registered
independent branch, provenance, or undo-cost axis.

The current priority rule is:

```text
keep detector provenance active;
keep weak measurement demoted
until a monitored platform names an independent D1 axis before analysis
and that axis changes the TaF verdict with standard monitored statistics fixed.
```

## Support Chain

- T83: detector-side Q1 remains non-null only as a pre-registered raw-log
  admissibility protocol, not a detector-dynamics law.
- T85: the detector route narrows further, but spoof/unique-tag evidence can
  still demote the signed fixture on its own.
- T86: perturbation-only and signed-DAG-only rescues survive in a hostile
  ambiguous-tag family, so the detector schema still has isolated positive
  witness shapes.
- T91: named weak-measurement platforms fail the independent-axis gate because
  their candidate TaF axes collapse to the same monitoring stream or to
  postselected recovery.
- T93: the remaining weak-measurement undo-cost escape hatch is only a witness
  shape, not a real platform. Control proxies and success-conditioned recovery
  are null.

Together these results create an asymmetry:

```text
detector provenance = blocked but still executable on named instrumentation
weak measurement = blocked before executable platform choice
```

That asymmetry is enough to reorder priorities.

## Current Strongest Claim

The strongest defensible Q1 strategy claim is now:

```text
T12 is no longer the active lead discriminator route. It is a reinstatement-
only branch. Detector provenance remains ahead until weak measurement supplies
a real pre-registered independent axis that survives same-record and
postselection collapse.
```

This is a strategic demotion, not a theoretical deletion. Weak measurement can
return, but only by clearing a stricter gate than detector provenance currently
faces.

## What This Improved

This report prevents wasted effort. Without an explicit demotion rule, future
runs can keep reopening homodyne, uncollapse, or jump-reversal examples even
though T91 and T93 already showed why those routes are null.

It also sharpens roadmap honesty. The active frontier is no longer "whichever
route sounds like the cleaner experiment." It is the route with the best
remaining executable falsification path.

## What This Weakened Or Falsified

This weakens the idea that T12 should still be treated as the project's default
high-upside experimental discriminator.

It also falsifies a softer escape hatch: weak measurement does not stay
high-priority merely because an abstract positive witness shape exists in T90 or
T93. A real platform must instantiate that shape before priority is restored.

## Falsification Condition

This report fails if a concrete monitored weak-measurement platform provides a
pre-registered branch/provenance/undo-cost observable that:

1. is not reconstructible from the standard monitoring stream;
2. does not rely on success-conditioned postselection; and
3. changes the TaF verdict while standard monitored statistics remain fixed.

If that happens, weak measurement should be reinstated immediately as a live
head-to-head route against detector provenance.

## Claim Ledger Update

Q1 should remain `partially_supported`, but the active roadmap should now state:

```text
detector provenance remains the primary executable Q1 route;
weak measurement is demoted below it until a real platform clears the
independent-axis gate.
```

## Open Blocker

Weak measurement still lacks the one object its route needs most: a named
platform with a pre-registered independent axis and a raw-log schema that can
be audited without collapsing back to the same standard monitored record.

Detector provenance still lacks the one object its route needs most: an actual
T78-style deployment with published event-level raw logs.

## Next Work

Stop platform-shopping inside standard weak measurement unless the candidate
starts by naming the independent axis and raw-log schema.

Spend the next Q1 experimental effort on one of two moves:

1. obtain a realistic detector deployment plan that can satisfy T78/T87 in
   practice; or
2. name a genuinely new monitored weak-measurement platform that clears T91 and
   T93 before any further modeling.
