# T111: D1 Gauge-Invariance Audit

## Route

Gauge-theoretic reformulation / GU mathematical-language import.

## Question

Which parts of the D1 finality profile are invariant, covariant, or
gauge-dependent under observer relabeling, record relabeling, and admissible
record-bundle automorphisms?

## Motivation

The finality gauge-theory open problem asks whether finality comparisons can be
treated as connection-like transport between observer frames. That question
cannot be attacked responsibly until the repo knows what transforms and what
is invariant.

The first bounded goal is therefore not a curvature computation. It is a
gauge-invariance audit of the D1 dimensions:

```text
accessible support
distinct-holder redundancy
causal branch support
graph reversal count
```

Only invariant or correctly covariant quantities should appear in any later
candidate physical observable, connection, curvature, or anomaly-style
condition.

## Construction

Define a finite action on D1 record systems generated by:

- observer relabeling;
- record-label permutation preserving source/target incidence;
- holder relabeling preserving independence partitions;
- causal-graph isomorphism preserving reachability;
- access-boundary refinement and coarsening maps;
- non-admissible controls that intentionally break one of the above
  preservation rules.

For each transformation, compute the D1 profile before and after transformation
and classify each dimension as:

```text
invariant
covariant with declared transport rule
gauge-dependent / not observable
undefined under this transformation
```

## Success Criteria

- Define the allowed transformation groupoid or finite action explicitly.
- Classify all four D1 dimensions under each allowed transformation.
- Separate relabeling invariance from access-boundary changes; the latter may
  be physical data, not gauge.
- Identify at least one negative control where a non-admissible transformation
  changes an allegedly invariant quantity.
- State which quantities can be used in a future finality connection.
- State which quantities must remain observer-frame data rather than physical
  observables.

## v0.1 Result

The finite audit is implemented. In the reference system the D1 profile is:

```text
(accessible support, distinct-holder redundancy, causal branch support, graph reversal count)
= (4, 2, 2, 2)
```

The pure gauge maps preserve all four components:

- observer relabeling;
- record-label permutation preserving incidence;
- holder relabeling preserving independence partitions;
- causal-graph isomorphism preserving reachability.

Access-boundary refinement and coarsening are classified as covariant, not
pure gauge. Refinement gives `(2, 2, 2, 0)` and coarsening gives
`(5, 3, 3, 3)`. These value changes are observer-frame boundary data, not
coordinate artifacts.

Negative controls break the required preservation clauses and are classified
as undefined:

- record incidence break changes the profile to `(4, 1, 1, 2)`;
- holder partition merge changes the profile to `(4, 1, 2, 2)`;
- causal non-isomorphism changes the profile to `(4, 2, 1, 2)`.

The conservative conclusion is that all four components are transportable as
boundary-indexed D1 coordinates under the stated finite maps. They are not one
access-independent physical scalar. Branch support and graph reversal count
remain formal-only in current physical claims.

## Failure Criteria

- It treats every observer change as gauge, erasing real access-boundary
  differences.
- It treats every profile change as physical, leaving no gauge redundancy.
- It assumes gauge invariance from notation rather than proving it under a
  stated action.
- It uses curvature, gravity, or anomaly language before the invariant
  quantities are identified.
- It changes Q1 or D1 claim status directly from this audit.

## Claim Impact

T111 supplies the entry condition for any later "finality connection" program.
It says which D1 data are transportable under pure relabeling maps and which
changes are real access-boundary data rather than gauge.

The gauge-theory and gravity branch remains open. T111 is an invariance gate,
not a connection, curvature, gravity, or anomaly result.

## Known Guardrails

- Gauge invariance is a technical condition, not a metaphor.
- Access restriction is often physical data in TaF, not a coordinate artifact.
- No curvature or gravity claim follows from this audit alone.
- No anomaly-cancellation claim follows unless the holder-count condition is
  also shown to transform correctly.
