# T115: Maintenance-Cost Viability Split

## Route

Thermodynamic arrow / mathematical machinery / prior-art absorption audit.

## Question

Can systems match on ordinary entropy, control, stability, viability,
resilience, and storage metrics while differing in future operation rights or
maintained record usability?

## Setup

Construct paired finite systems. Each pair has the same:

- entropy/export proxy;
- control effort proxy;
- stability window;
- viability-kernel label;
- resilience recovery steps;
- storage bits;
- maintained record count.

The pair is then scored on:

- accessible witnesses;
- required witnesses;
- operation rights;
- reconstruction paths;
- admissibility tokens.

## Required Toy Domains

- Record commons.
- Provenance.
- Version control.
- Governance.
- Consensus.

## Success Criteria

- At least one matched-standard pair splits on future operation capability.
- The split is not driven by entropy, control effort, stability, resilience,
  storage, or ordinary viability labels.
- The result explicitly compares against viability kernels, free-energy
  formulations, resilience/adaptive-cycle models, commons/public-goods models,
  control-theoretic state maintenance, information-theoretic storage,
  provenance systems, and LossKernel/reconstruction debt.

## Failure Criteria

- Future operation capability is assigned by intuition rather than finite
  witness/rights/admissibility fields.
- The model treats "maintenance cost matters" as a positive result.
- The separation disappears once ordinary provenance, commons governance,
  version-control history, or reconstruction-debt state is included.
- The artifact promotes the maintenance-cost direction as new physics.

## Claim Impact

T115 does not upgrade any core claim. It preserves maintenance-cost viability
only as a narrow operation-right/provenance/reconstruction-debt audit object.
If future runs cannot find a non-engineered case that survives enriched
existing theories, the direction should be kept as organizing vocabulary.
