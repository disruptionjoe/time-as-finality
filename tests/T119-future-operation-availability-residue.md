# T119: Future Operation Availability Residue Audit

## Route

Mathematical machinery / adoption elsewhere.

## Question

Are Q1A, ASP, reconstruction debt, maintenance cost, provenance, LossKernel,
and admissibility work converging on the same object, or are they merely using
similar future-action language?

## Candidate Object

Future Operation Availability (FOA):

```text
FOA(O, T, h) = the set of future operations in task universe T that remain
available to observer/system O over horizon h under witness, right,
reconstruction, certification, and maintenance constraints.
```

## Success Criteria

- Audit the named branches for what collapsed and what survived.
- Check whether the surviving object really involves FOA.
- Apply prior-art pressure from viability, reachability, affordance, active
  inference, RL, opportunity-set economics, capability theory, mechanism
  design, commons governance, and control theory.
- Construct three finite examples:
  - FOA differs while existing measures do not.
  - Existing measures differ while FOA does not.
  - FOA collapses into an existing framework.
- Recommend reject, preserve, formalize, or promote.

## Failure Criteria

- Treating any future-facing phrase as FOA without declaring observer, task
  universe, horizon, witnesses, rights, reconstruction paths, certifications,
  and maintenance constraints.
- Claiming novelty against coarse versions of prior art while enriched prior
  art absorbs the object.
- Promoting FOA without a finite separation from enriched reachability or
  opportunity-set formalisms.

## Claim Impact

If FOA survives, it should be preserved only as a cross-branch audit normal
form unless it separates from enriched prior art. If it does not survive, the
recurrence should be treated as a framing artifact.
