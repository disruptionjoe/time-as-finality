## Setup

T17 builds a finite local-update system with environment cells, sensor cells,
persistent memory cells, written flags, and an internal comparator.

The point is to remove one major external assumption from earlier tests:
instead of selecting a terminal observer window and then asking what records
are accessible there, the model creates a small reconciler subsystem whose
storage, access boundary, and comparison state arise inside the dynamics.

The state is:

```text
(env_a, env_b, sensor_a, sensor_b, memory_a, memory_b,
 written_a, written_b, compared, decision)
```

Local updates:

1. sensors copy coupled environment cells;
2. memory cells latch the first sensed value and persist;
3. written flags distinguish stored false from unwritten;
4. the comparator activates after both memory cells have written;
5. the decision bit is true when the stored memory values agree.

## Success Criteria

The model succeeds if:

- both true-valued and false-valued records can persist;
- the access boundary is generated by internal wiring, not selected as a final
  observer window;
- the comparator reconciles at least two stored records;
- equal and unequal cases both produce internal decisions;
- intervention on either environment input changes the internal decision while
  preserving the same generated boundary;
- a no-persistence control has no records at the final state.

## Failure Criteria

The model fails if:

- false-valued records are indistinguishable from absent records;
- the decision depends on externally reading a terminal window instead of an
  internal comparator state;
- access holders must be hand-selected after the run;
- the no-persistence control still produces final records;
- consciousness, phenomenal experience, or a full physical observer is
  inferred from the toy reconciler.

## Known Physics Constraints

This is a finite classical local-update witness. It does not derive quantum
measurement, thermodynamic irreversibility, consciousness, proper time, or a
physical observer from first principles.

## Contribution Needed

Extend this from a two-record latch to a generated record network where
records can be overwritten, degraded, copied, contradicted, or reconciled
under bounded energy or channel constraints.
