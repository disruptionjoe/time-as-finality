# T381: Observer Relativity Necessity Screen

## Target Claims

R1, S1, D1, PO1, and the T380 live edge:

```text
Is a minimal two-null-channel adapter necessary for observer-relativity behavior,
or are there simpler/alternative adapter shapes that satisfy the same needs?
```

This is a finite requirement screen. It is not a proof from arbitrary
compatibility structure.

## Setup

Evaluate candidate adapter shapes against the requirements used by T379/T380:

- source-compatible nonnegative compatibility counts,
- two independent signal directions in 1+1,
- exactly two primitive rays for minimality,
- unit-normalized primitive one-step representatives,
- one nondegenerate forced interval form,
- nontrivial reciprocal observer scaling,
- simultaneity split under observer change.

Candidates:

- one signal channel,
- two collinear signal labels,
- minimal two-null-channel basis,
- overcomplete three-channel interface,
- asymmetric-unit two-channel interface,
- signed diagonal two-channel interface.

## Success Criteria

- The minimal two-null-channel adapter is the unique survivor.
- One-channel and collinear controls fail directionality.
- Three-channel overcomplete control fails minimality / interval forcing.
- Asymmetric-unit two-channel control reduces to the same two null rays but is
  not minimal under unit normalization.
- Signed diagonal control is rejected as non-source-compatible.

## Failure Criteria

- A one-channel or collinear adapter survives.
- A three-noncollinear primitive adapter survives as minimal.
- The result claims compatibility alone derives the adapter basis.

## Known Constraints

The screen bakes in the observer-relativity requirements. It shows necessity
inside that declared space, not from all possible compatibility structures.

## Contribution Needed

The next step is to classify whether this surviving two-null-channel shape is
best read as an internal compatibility derivation or as an external adapter
interface.
