## The question this test settles

The adopted capability-boundary taxonomy says that in the classical finite-state
regime, exact single-instance physical boundaries do not appear: a separator that
is outside the declared region algebra is still recovered by the full co-present
finite code once A1 enlargement is admitted.

T432 asks:

```text
Can the classical finite-state fragment be made executable as a gate: any
single-instance separator outside A0 but defined on the finite closed product code
is E0-declared relative to A1?
```

## Certificate inputs

The gate uses finite Boolean product configurations.

- `A0`: observables generated by the declared region coordinates.
- `A1`: the full co-present finite classical code.
- Separator: a binary datum on configurations.

It includes named certificates plus an exhaustive small Boolean sweep over all
proper regions and all binary datums for `n in {1,2,3}`.

## Success / Kill Criteria

### Success

For every tested boundary datum:

- the datum is not determined by A0;
- a same-A0 pair has different datum values;
- the datum is determined by A1 full-code lookup;
- the single-instance verdict is `E0_DECLARED_RELATIVE_TO_A1_FULL_CODE`;
- no physical candidate appears in the exhaustive small sweep.

### Guard case

At least one certificate must have no proper coordinate support and still be
A1-declared. This prevents the gate from proving only an easy complement-bit case.

### REDESIGN

If any finite classical single-instance separator is not recovered by the full
code, stop and inspect the model: either A1 was misdefined or the classical
fragment has been misstated.
