# T500: Competency Resource Permission Stack Gate

## Target Claims

Composite absorber-stack progress lanes; C(R) capability object; T493/T494
competency absorber line; region-indexed capability discriminator. No
claim-ledger target.

## Setup

Use the fourth composite absorber-stack lane:

```text
full task-success competency profile
+ resource preorder
+ boundary/permission completion
+ provenance completion
```

The executable gate asks whether any C(R)-style residual survives after all
four absorber layers are granted. It distinguishes:

1. Current T493/T494 C(R) material absorbed as a composite operational profile.
2. Weak one-layer readings that do not carry the full C(R) burden.
3. A synthetic future packet that grants the full stack and still reports
   controlled capability spread.
4. Claim, public-posture, mechanism-import, external-action, and cross-repo
   shortcuts.

## Success Criteria

- The current T493/T494 C(R) reading is absorbed once the full stack is granted.
- Single-statistic, resource-only, permission-only, and provenance-only packets
  are rejected as incomplete.
- A synthetic packet with residual spread after the full stack is admitted only
  as a future review target.
- Missing region/menu/task context, external mechanism import, claim/public
  posture, external publication, and cross-repo shortcuts are rejected or
  blocked.
- No claim-ledger, roadmap, README, North Star, public-posture, hard-policy,
  protected-license, external-publication, or cross-repo truth movement occurs.

## Failure Criteria

- T500 treats C(R) as a new primitive after the stack is granted.
- T500 promotes a region-indexed discriminator success.
- T500 uses competency/resource/permission/provenance language to move claims
  or public posture.
- T500 imports Levin/Fields, active-inference, free-energy, or polycomputing
  mechanism language into TaF.
- T500 writes outside this repository or requires non-GitHub external action.

## Known Physics Constraints

This is not a physics result. It does not assert a new physical capability
object, a biological mechanism, a resource-theory theorem, or any public-facing
physics claim.

## Contribution Needed

Use T500 as the default C(R)-stack screen before reopening the region-indexed
capability discriminator. Future packets should show non-singleton capability
spread after the full competency/resource/permission/provenance stack is
declared, or demote to a composite explanation.
