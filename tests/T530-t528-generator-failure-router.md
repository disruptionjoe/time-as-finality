# T530: T528 Generator Failure Router

## Target Claims

- [S1: Spacetime As Consensus Envelope](../claims/S1-spacetime-consensus-envelope.md)
- [T525: Repaired S1 Manifoldlikeness Suite](T525-repaired-s1-manifoldlikeness-suite.md)
- [T526: S1 Reference-Law Gap Audit](T526-s1-reference-law-gap-audit.md)
- [T528: S1 Finality-Native Generator Preflight](T528-s1-finality-native-generator-preflight.md)

## Central Question

After T528 rejects the first two-receipt finality-native generator packet, what
exactly failed, and which next packet shapes are admissible for review without
screen-conditioning, Lorentzian import, or S1 posture movement?

## Setup

T530 consumes the executed T528 audits and classifies the failed samples by the
repaired-suite axes:

- hard T126 gate;
- ordering fraction;
- height;
- width;
- largest interval.

It then routes candidate next packet shapes:

- axis-blind receipt-count tweaks;
- screen-tuned mixture laws;
- Lorentzian-reference reuse;
- height-only repairs;
- ordering-fraction repair without independent naturality;
- predeclared finality-native measure-law packets that target the primary
  failure axis and supply independent naturality plus hostile controls.

## Success Criteria

- The router reports T528's 25/32 pass rate and 7 failed samples.
- The primary failure axis is identified as ordering fraction, not a hard-gate
  rejection.
- Height is retained as a secondary failure axis.
- Screen-conditioned and Lorentzian-import shortcuts are rejected or blocked.
- Only a future predeclared finality-native packet that targets the primary
  failure axis, supplies independent naturality or neighbor-theory support, and
  includes hostile controls is admitted as a review target.
- The result counts as no S1 evidence and changes no claim/canon/public posture.

## Failure Criteria

T530 fails if:

- it treats T528's incomplete pass as S1 evidence;
- it hides which repaired-suite axes failed;
- it recommends a screen-conditioned or imported-coordinate law;
- it admits a packet that does not address the primary failure axis;
- it changes S1 status, claim status, Canon Index tiers, canon verdicts, public
  posture, external-publication state, or cross-repo truth.

## Run Command

```bash
python -m models.run_t530

```

## Boundary

T530 is a review-only router over an already rejected preflight. It does not
derive spacetime, prove manifoldlikeness, establish a finality-native
sprinkling law, repair T528, reverse T223, promote S1, change claim status,
change canon, change public posture, or authorize external publication.
