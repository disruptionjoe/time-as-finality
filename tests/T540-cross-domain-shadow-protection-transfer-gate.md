# T540: Cross-Domain Shadow-Protection Transfer Gate

## Target Claims

- TAF8: Cross-domain shadow-protection theorem target.
- [T492: Typed Gap Category Bridge](T492-typed-gap-category-bridge.md)
- [T499: Kappa Composite Residual Template Gate](T499-kappa-composite-residual-template-gate.md)
- [T501: Typed Translation Object-Identity Stack Gate](T501-typed-translation-object-identity-stack-gate.md)

## Central Question

Do the current kappa and typed-gap catalogues already clear a reusable
cross-domain shadow-protection theorem burden without identity by construction?

T540 consumes the committed T492/T499/T501 ceilings and turns TAF8 into an
executable transfer gate. It checks whether the current catalogues are theorem
evidence or only a scaffold for the next nonidentity packet.

## Success Criteria

- The computation is deterministic and exits 0.
- T499 is consumed as method-template only, not kappa promotion.
- T501 is consumed as schema-preservation only, not object identity.
- The current catalogue pair passes the reusable TAF8 scaffold but fails theorem
  preconditions where nonidentity and direct preservation burdens remain open.
- Identity-by-construction, per-domain retuning, missing native controls, and
  claim/public-posture shortcuts reject or block.
- A synthetic full-burden packet is admitted only as a future review target.
- No claim-ledger, Canon Index, canon-verdict, public-posture,
  external-publication, or cross-repo truth movement occurs.

## Failure Criteria

T540 fails if:

- it treats shared schema or shared rank as object identity;
- it lets the current catalogue pair count as a theorem;
- it promotes kappa, typed-gap category language, physics language, or TAF8
  public posture;
- it uses per-domain retuning as a cross-domain theorem spine;
- it omits positive controls, negative fixtures, or absorber completion;
- it moves claim status, Canon Index tiers, canon verdicts, public posture,
  external-publication state, or cross-repo truth.

## Run Command

```bash
python -m models.t540_cross_domain_shadow_protection_transfer_gate --write-results
python -m unittest tests.test_t540_cross_domain_shadow_protection_transfer_gate -v
```

## Boundary

T540 is a TAF8 transfer-precondition gate. It does not prove a cross-domain
theorem, promote a claim, or update any governance or public posture surface.
