# T8: Observer Renderer Toy Model

## Target Claims

- [Consciousness As Record Renderer](../open-problems/consciousness-as-record-renderer.md)
- [D2: Observer As Record-Bearing System](../claims/D2-observer-as-record-bearing-system.md)
- [G3: Observer Rendering Is Not Mind-Created Matter](../guardrails/G3-observer-rendering-not-mind-created-matter.md)

## Setup

Build a minimal toy model with two observer-renderers and one shared artifact.

Each observer has:

- memory records;
- procedural scripts or habits;
- sensory records;
- attention or working context;
- accessible physical/social records;
- a rendered body-interface through which actions and constraints are experienced;
- a rendered world state.

The model should show how an observer's rendered world updates after:

- new sensory input;
- memory correction;
- interaction with another observer;
- creation or alteration of a shared artifact;
- contradiction between private memory and external record.

## Success Criteria

- The model represents the body as a high-finality participation interface, not as the deepest observer.
- The model distinguishes private memory, physical record, social record, and shared artifact.
- The model can represent another observer as independently real but only indirectly accessible.
- The model shows shared finality increasing after record exchange or artifact creation.
- The model does not imply that subjective belief creates matter.

## Failure Criteria

- The model cannot distinguish hallucination from corroborated record.
- The model treats the rendered body as arbitrary or freely rewriteable.
- The model treats other observers as NPCs or purely private representations.
- The model requires physical law to be generated by personal belief.
- The model adds no clarity beyond ordinary cognitive-science language.

## Contribution Needed

Define a small state-transition model:

```text
Observer O = (R, P, A, W, B_i)

R = accessible records
P = procedures / scripts / habits
A = attention or active context
W = rendered world state
B_i = rendered body-interface

Interaction:
  O1.W + O2.W + artifact record
    -> record exchange
    -> consistency / conflict check
    -> updated W1, W2
    -> increased or decreased shared finality
```

Then identify which parts are cognitive modeling, which parts are social-finality modeling, and which parts are only metaphor.
