# Information Portfolio — Critique Log

## Draft → Pass A (adversarial)
- **Core risk: confusing failure with waste.** Operating model §3 is emphatic.
  Made archival-gain-capture a first-class, must-fire path; every retirement yields
  a positive-framed entry. Added it as both a procedure priority and a Phase-4
  wiring coverage question.
- **Authority creep via the balance read.** A balance read could be (mis)used to
  reallocate budget. Constrained it to *descriptive only*; target ratios and any
  reallocation route to decision-review / portfolio-review.
- **Status authorship.** A line's status must be read, not decided. Made status
  read-from-registry explicit in the entry schema and procedure; lifecycle moves
  route to lifecycle-review (DEC-007/018).

## Revise → Pass B (adversarial)

  score; mirrored it — qualitative what-changed + gain-type tag, no scalar, no
  cross-line ranking. Added as a Must-not and a failure mode.
- **Double-counting gain.** Added a dedup/coverage check citing specific runs.
- **Restating authoritative status (DEC-007).** Explicit Must-not + failure-mode

- **Routing existence.** lifecycle-review + line-registry (read) are locked/exist;
  portfolio-review + decision-review are planned this run; research-memory handles
