# Contradiction Hunter Manual Run

## Scope

This run tested the strongest current claim cluster visible from the North Star
map, workflow registry, claim ledger, and recent persona artifacts:

- TaF may be a concrete lead line for testing observer-shadow geometry.
- Capability projection, record finality, and observer-visible invariants may
  expose something deeper than a local metaphor.
- Recent residues around representation invariance, safe conditioning,
  LossKernel classification, and motif emergence may be mutually reinforcing.

The run deliberately did not ask whether the vision is inspiring. It asked what
would make the claim false, overstated, circular, or merely verbal.

## Critic Packet

Each critic was given only a narrow packet:

- the claim cluster above;
- the supporting orientation docs and recent persona-run conclusions;
- the minimum glossary needed to parse capability projection, observer access,
  finality, witness, and absorber language.

They were not given a broad defense of the project.

## Independent Critiques

| Critic | Main attack | Required repair |
|---|---|---|
| C1 | The repo may be accumulating disciplined analogies without a single typed source object or observer-shadow projection forced by known theory. | Produce one concrete typed bridge where a known native theory induces `Cap`, an observer projection, invariance requirements, and absorber-resistant residue. |
| C2 | The vocabulary is not yet discriminating unless it predicts concrete failures that rival framings would not predict. | Define one operational test with plausible TaF-negative outcomes against reconstruction, admissibility, forgetting, or future-operation constraints. |
| C3 | The methodology can validate itself by producing more workflows rather than forcing specific failures. | Run one capability-projection test with fixed horizon, boundary, comparison, and demotion rule. |

## Converged Weak Point

All three critics independently attacked the same structural weakness:

> The current high-level synthesis is too close to a disciplined vocabulary
> unless it forces one typed, falsifiable capability-projection test.

This is not a rejection of the project. It is a priority proof obligation.

## Priority Task

Create a single typed capability-projection test with the following fields:

| Field | Requirement |
|---|---|
| `Y` | Source state or native object from a specific theory or toy model. |
| `O` | Observer, access boundary, or measurement regime. |
| `pi` | Projection from source state to observer-visible record. |
| `X` | Visible state, record, or shadow after projection. |
| `T, h, B` | Task, horizon, and boundary condition. |
| `Cap` | Capability or future operation whose preservation is being tested. |
| `R_K` | Native comparison relation from a rival or absorbing framework. |
| `W` | Witness, certificate, or provenance object, if one is claimed. |
| Pass condition | Capability factors through the visible record with explicit residual limits. |
| Fail condition | Capability cannot be recovered, is absorbed by the native theory, or depends on hidden global structure. |
| Demotion | If fail, weaken geometry-shadow language to local audit vocabulary for this case. |

## Claim Weakening Rule

Until this task exists, the strongest safe statement is:

> TaF currently supplies a disciplined audit vocabulary for observer-indexed
> records, projections, and future-operation availability. It is not yet a
> demonstrated geometry whose shadows include physics.

If the typed capability-projection test fails, weaken further:

> TaF is a local research method for classifying records, witnesses, and access
> failures, not evidence for a shared observer-shadow geometry.

## Handoff

The next workflow should not expand the claim set. It should look for recurring
motifs that can be turned into the typed test fields above.
