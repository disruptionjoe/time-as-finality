## Genuine Shared Structure

The strongest bridge is not "these projects use the same words." It is:

> Each project studies when a downstream judgment is admissible after access has
> been constrained by a boundary, projection, protocol, or class of allowed
> transformations.

Concrete parallels:

| Abstract operation | TaF instance | Legitimacy instance | GU instance |
|---|---|---|---|
| Declare access boundary | Observer-visible record | Visible contribution/rule record | Six-axis observer leg |
| Preserve future operation | Capability after finality | Retained rights, appeal, recognition | Smooth-shadow or signed-readout interpretation |
| Attach witness/certificate | Provenance, detector record, representation certificate | Contribution log, rationale, validation path | Specification before advocacy; finite control case |
| Test admissibility | Absorber/demotion pass | Contestability and capture review | No-go assumption and class-exit discipline |
| Handle failure | Weaken claim, expose residue | Appeal, rule change, exit with retained record | Demote exploratory branch or fail control case |

This suggests a useful bridge object:

```text
admissibility package =
  visible record
  + declared authority/access boundary
  + witness or certificate path
  + allowed transformations
  + failure and contestation rule
```

## Tempting But False Analogy

The following analogies must be treated as unsafe:

- Record finality is not the same thing as legal or institutional legitimacy.
- Legal settlement is not the same thing as physical invariance.
- Gauge-invariant or no-go-relative structure is not evidence for social
  legitimacy.
- A contribution log is not automatically a witness in the mathematical sense.
- TaF finality is not a physics proof for GU.

The GU crosswalk explicitly forbids treating finality as a new physics proof,
using observer agreement as a replacement for anomaly or spectral-triple work,
or importing hidden global commit order into relativity-facing models.

## Bridge Test

Create a three-repo operation crosswalk. Each proposed bridge row must fill:

- source object;
- observer/access boundary;
- admissible transformations;
- comparison relation;
- witness or certificate object;
- authority or rule-change locus, if any;
- native failure mode;
- absorber that could explain the structure without the bridge.

If a row cannot fill these fields without metaphor, the bridge demotes.

## Best First Artifact

Build a small table for one operation only:

Then instantiate it in:

The point is not to prove unity. The point is to see whether the same operation
schema can survive three native failure modes.

## Verdict

The bridge is promising as an operation-level comparison. It should not be
promoted as a shared theory until the bridge test survives negative controls in
all three repositories.
