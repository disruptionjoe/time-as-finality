# Extension Workflow Improvement Considerations

## Review

The five workflows are runnable as human-readable protocols. Sequentially, they
formed a useful loop:

1. Contradiction Hunter forced a priority proof obligation instead of producing
   generic critique.
2. Motif Census detected a recurring admissibility-object structure.
3. Theory Compression showed that observer-indexed record sufficiency can
   regenerate many live tasks.
4. Cross-Repo Bridge Builder found a real operation-level bridge while
   preserving negative controls.
5. Theory Tournament selected a lead interpretation without declaring it true.

The strongest result of the sequence is this routing decision:

> The next concrete task should be a typed capability-projection test centered
> on admissible witness-carrying records.

The strongest methodological result is that the workflows work best as a chain,
not as independent artifact factories.

## Gaps Found

| Gap | Impact |
|---|---|
| No shared run-log template | Outputs can drift in structure and quality across runs. |
| No packet templates for spawned agents | Fresh-context critics and specialists may receive inconsistent context. |
| No default corpus selection rule | Runs can sprawl or cherry-pick recent artifacts. |
| Weak handoff schema between workflows | Useful outputs may not become tasks for the next workflow. |
| Cross-repo bridge has no external-context standard | Bridge quality depends too much on ad hoc doc selection. |
| No demotion rule for workflow outputs themselves | A weak motif map or bridge memo can look authoritative just because it exists. |
| Automation coverage is intentionally absent | The workflows are added as protocols but not yet integrated into task routing. |

## Recommended Improvements

### P0: Add Shared Templates

Add reusable templates for:

- run metadata;
- corpus packet;
- critic packet;
- motif-specialist packet;
- compression packet;
- bridge source packet;
- tournament scorecard;
- handoff task.

This is the highest-leverage improvement because it makes future runs comparable
without changing automation behavior.

### P0: Define Workflow Handoff Objects

Each workflow should emit one typed handoff object:

| Workflow | Handoff object |
|---|---|
| Contradiction Hunter | `proof_obligation` or `weakening_rule` |
| Motif Census | `motif_definition_candidate` |
| Theory Compression | `candidate_essence` plus `lost_structure` |
| Cross-Repo Bridge Builder | `bridge_test` plus `false_analogy` |
| Theory Tournament | `lead_interpretation` plus `minority_programs` |

This prevents the sequence from becoming five separate essays.

### P1: Add A Sequence Orchestrator Document

Add a short protocol that defines the default chain:

```text
contradiction -> motif census -> compression -> bridge -> tournament -> improvement review
```

It should specify when to stop early, when to demote, and when to open a new
task instead of continuing to synthesize.

### P1: Standardize Cross-Repo Negative Controls

Bridge runs should require each repo to supply:

- native success condition;
- native failure condition;
- native absorber;
- forbidden analogy;
- minimum docs read.

This is necessary because bridge work is the easiest place for useful
comparison to become vague megatheory.

### P2: Consider Automation Later

Do not wire these into automation triggers yet. They need two or three manual
runs first so the packet and handoff schemas stabilize. Premature automation
would amplify inconsistent output.

## First Concrete Follow-Up

Create a bounded task:

```text
Define and test an admissible witness-carrying record for one typed
capability-projection example.
```

Required fields:

- source object;
- observer/access boundary;
- projection;
- visible record;
- future operation;
- witness/certificate;
- comparison relation;
- absorber;
- pass/fail condition;
- demotion rule.

## Verdict

The workflows are worth keeping. Their main improvement need is not more
ambition. It is stricter input packets, explicit handoff objects, and a shared
standard for when an output weakens the project instead of adding another
artifact.
