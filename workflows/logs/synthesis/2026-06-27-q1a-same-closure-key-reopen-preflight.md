## Frozen Same-Closure-Key Burden

A proposed Q1A reopen packet is admissible for review only if it freezes all of
the following before any verdict split is claimed:

1. Standard quantum-side summaries:
   - decoherence and pointer-basis evidence;
   - fragment-information summaries;
   - ordinary branch or history availability.
2. The audited provenance-aware fragment partition.
3. The access cut and partition visibility.
4. The audited accessible provenance-support count.
5. The D1 verdict map being tested.
6. The neighboring framework package being granted the same audited partition,
   access cut, and support count.
7. The candidate physical dimension expected to split the D1 verdict.
8. The demotion rule for showing that the candidate dimension collapses back to
   the existing closure key.

If any of these are missing, the packet is bookkeeping-by-default and should not
be routed into autonomous Q1A toy-model work.

## Admissible Reopen Routes

### Route A: Same-Closure-Key Verdict Split

The candidate must hold fixed the standard quantum summaries, audited partition,
access cut, partition visibility, and audited accessible provenance-support
count. It must then change the D1 verdict through a predeclared physical
dimension that is not equivalent to any of the following:

- raw fragment count;
- relabeled branch or history variation;
- changed access cut;
- hidden partition mutation;
- reversal, erasure, or undo language that reduces to support counting;
- observer-relative prose already absorbable by RQM, QBism, Everett-style
  bookkeeping, decoherence, Quantum Darwinism, or consistent histories.

The split must be same-key in the strict sense: if the closure-key values differ,
the run is testing a changed accounting package, not a Q1A reopen route.

### Route B: Physical Partition-Rule Derivation

The candidate may instead derive the admissible partition or threshold rule from
additional physical structure. This route is admissible only if the proposal:

1. Names the added physical structure before scoring examples.
2. States the observable or operational constraint that fixes the partition
   rule.
3. Shows why a neighboring framework cannot import the same rule without adding
   the same new physics.
4. Includes a hostile absorber check granting the rule to decoherence, Quantum
   Darwinism, consistent histories, RQM, QBism, and Everett-style bookkeeping.
5. Demotes itself if the rule becomes ordinary provenance-aware record
   accounting once the neighbor is granted the same information.

A better story about records is not enough. The route needs either a same-key
verdict split or an extra physical rule that does not become neighbor-owned as
soon as it is made explicit.

## Null And Demotion Conditions

Demote the candidate to `bookkeeping_only` or `bookkeeping_by_default` if any of
the following occur:

- It does not freeze the standard quantum-side summaries before the verdict.
- It changes decoherence, fragment data, branch/history data, partition
  visibility, access cut, or audited support while claiming same-key status.
- It reintroduces raw redundancy under a new label.
- It hides a changed provenance partition inside the example description.
- It uses reversal, erasure, or undo cost without a calibrated observable that
  survives the T118 closure.
- It treats "observer-relative" language alone as a distinction from existing
  interpretive or record-accounting frameworks.
- It withholds the audited partition or access cut from the neighbor and calls
  the resulting split Q1A evidence.
- It derives a partition rule that the neighbor can import without adding new
  physics.
- It supplies another closed-family toy model rather than a same-closure-key
  physical split or a non-importable physical partition rule.
