# Rendered Interface Assumptions Preflight

## Scope

This note completes fourth-batch task 10 as a synthesis/preflight artifact. It
does not promote S1, replace relativity, solve black-hole information, or edit
canonical repo surfaces.

The purpose is to turn rendered-interface language into an assumption ledger:
define source, observer-protocol, and classical-record layers; declare what the
source-to-interface map preserves; and name failure conditions before any
proposal uses a deeper layer to evade no-go constraints.

## Layer Ledger To Freeze

A rendered-interface proposal is admissible for a next run only if it freezes:

```text
RenderedInterfacePacket = (
  source_layer_S,
  observer_protocol_layer_P,
  classical_record_layer_C,
  source_to_protocol_map_m_SP,
  protocol_to_record_map_m_PC,
  preserved_assumptions,
  weakened_assumptions,
  lost_assumptions,
  failure_conditions
)
```

Minimum layer meanings:

| Layer | Required declaration |
| --- | --- |
| `source_layer_S` | Deeper structure, event set, constraint system, or pre-interface substrate being proposed. |
| `observer_protocol_layer_P` | Access channels, observer worldline or access events, direct participation, received signals, indirect channels, and boundary permeability. |
| `classical_record_layer_C` | Record tokens, local histories, compatibility relation, and observer-relative finality domains. |

Minimum preservation table:

```text
causal_reachability:
local_overlap_consistency:
no_global_commit_order:
remote_received_signal_access:
direct_participation_distinction:
horizon_direct_classical_export_limit:
metric_or_lorentzian_constraints:
domain_of_dependence_constraints:
encoded_or_indirect_channel_typing:
```

Any field marked "lost" must trigger a failure or a narrowed non-physics
proposal unless the run supplies a replacement constraint of equal strength.

## Local Consistency Gate

Acceptable target shape:

```text
overlapping causal regions
  -> mutually accessible records
  -> locally reconciled histories
  -> domain-relative finality
```

Rejected target shape:

```text
one global renderer
  -> one global synchronized state
  -> universal observer-independent commit order
```

The run must distinguish:

- remote observation through received causal records;
- direct local participation in a region;
- inaccessible events whose records cannot reach the observer;
- later reconciliation inside a common causal future;
- encoded or indirect channels that require native physics before judging
  recoverability.

## Preflight Protocol

1. Freeze the source, protocol, and classical-record layers.
2. Freeze the maps `m_SP` and `m_PC` before checking any no-go or horizon case.
3. Fill the preservation/weakened/lost assumption table.
4. Run the required controls:

| Control | Required behavior |
| --- | --- |
| Remote signal | Received photons or signals count as causal records. |
| Direct participation split | Direct local participation is distinguished from remote received-record access. |
| Spacelike pair | No invariant global order is assigned; later common-future reconciliation is allowed. |
| Horizon direct-classical cap | Exterior direct classical export is limited without treating interiors as unreal. |
| Encoded/indirect channel | Withheld unless native physics and channel type are declared. |
| Global-renderer control | Rejected as hidden global synchronization. |
| No-go escape control | Rejected if the source map drops a no-go assumption without declaring the loss. |

5. Classify the result with one allowed verdict.

Allowed verdicts:

```text
assumption_ledger_admissible
local_consistency_only
standard_causal_access_absorption
underdeclared_source_map
global_renderer_null
no_go_escape_hatch_null
```

## Acceptance Criteria

The next run is decision-grade only if all of the following hold:

- `source_layer_S`, `observer_protocol_layer_P`, and
  `classical_record_layer_C` are explicitly defined.
- The maps from source to protocol to record layer are declared before any
  result is interpreted.
- The preservation table marks which assumptions are preserved, weakened, or
  lost.
- Local consistency is separated from global consensus.
- Remote astronomical observation through received signals is accepted as
  causal record access.
- Direct participation is not collapsed into received-record access.
- Event horizons are treated as causal-access boundaries, not transportation
  hubs or render gates.
- Encoded, holographic, complementarity, Page-curve, or quantum-extremal-surface
  questions are withheld unless native physics is typed.
- The result has one of the allowed verdicts above.

## Null Or Demotion Conditions

Treat the rendered-interface route as metaphor-only or null if any of the
following occur:

- The source layer is undefined or shifts after a no-go constraint is raised.
- The protocol layer lacks observer access events, record channels, or boundary
  permeability data.
- The classical record layer is only "what appears" with no record tokens or
  compatibility relation.
- The proposal denies ordinary remote observation by received photons or
  signals.
- Remote galaxies are called unrendered merely because the observer cannot
  travel there.
- A global renderer or universal synchronized commit order is introduced.
- Horizon language is used for travel, destination selection, or galaxy-to-galaxy
  render gates without a separate physics-compatible mechanism.
- The source map drops relativity, metric, causal, or no-signalling assumptions
  without declaring the loss.
- The proposal says "the deeper layer is different" as a no-go escape without a
  preservation table.
- A changed causal past, access diamond, or domain-of-dependence input is
  mistaken for independent S1 evidence.

Null result language to preserve:

```text
The rendered-interface proposal did not clear the assumption ledger. The safe
residue is local causal-record consistency language, not a global renderer, not
a spacetime derivation, and not an escape from native physics constraints.
```

  which assumptions are preserved, weakened, or lost.
- Do not edit `README`, `CLAIM-LEDGER`, `ROADMAP`, tests, models, results, or
  open-problem files from this preflight.
